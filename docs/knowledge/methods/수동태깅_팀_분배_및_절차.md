## 4. 진행 절차 — 단계별

### Step 1 — 환경 준비 (10분, 모든 팀원)

- [ ] 본인 분담 차원 확인 (위 §1 표)
- [ ] [manual_tagging_guide.md](manual_tagging_guide.md) 본인 차원 섹션 정독 (15분 이내)
- [ ] CSV 템플릿 열기: `data/processed/manual_tags_template.csv` (Excel/Google Sheets)
  - 행: 13 SKU
  - 열: 39 차원 — 본인 담당 컬럼만 채움. 나머지는 `N/A` 또는 빈칸


### Step 2 — 페이지 raw HTML fetch (박지윤·이소현)

**박지윤** (의료 9 페이지):
```bash
# 자사몰 fetch — curl_cffi (Cloudflare 우회)
python3 crawler/scripts/fetch_competitor_pages.py --vertical medical_device --brands bodydoctor,easyk,coway_therasol,ceragem_innerfit,elvie,perifit,kegel8,kgoal,peroneun
# → data/raw/medical_device/<brand>/2026-04-30/index.html 저장
```

**이소현** (가글 4 페이지):
```bash
python3 crawler/scripts/fetch_competitor_pages.py --vertical gargle --brands propolinse,listerine,garglin,perio
```

> 스크립트가 없으면 데마 팀에 요청 (curl_cffi Chrome impersonate 메모리 룰 준수). raw HTML이 안 깨끗하면 Playwright fallback.


### Step 4 — 본격 태깅 

각자 분담 차원 × 13(또는 9·4) SKU 채워 넣기.

- 의심 가는 케이스는 cell에 표시 (검수 우선순위)

### Step 5 — 자체 검토 

- [ ] 본인 채운 cell 일관성 체크 (예: 같은 SKU에서 T1 TABLE인데 T4 LOW면 자가모순?)

### Step 6 — 이중 태깅 (15분, 7명 모두)

> Cohen's kappa ≥ 0.6 목표 (manual_tagging_guide.md §F). 일치도 측정용.

**핵심 차원 cross-rater 검정** — 모든 팀원이 *자기 분담 차원 외에* 핵심 P0 차원 2개의 같은 SKU 2개를 독립 태깅:

| 팀원 | 본인 분담 차원 | 추가 cross 차원 (이중 태깅용) |
|:-:|---|---|
| 박지윤 (M1-M7) | M | T1 (주도 포맷) + Q1 (콘텐츠 깊이) — 의료 SKU 2개 |
| 이소현 (G1-G7+) | G | T1 + Q1 — 가글 SKU 2개 |
| 이경민 (T1-T5) | T1-5 | T6 (메시지 톤) + Q1 — 의료 SKU 2개 |
| 박재후 (T6-T10) | T6-10 | T1 + Q1 — 의료 SKU 2개 |
| 방우식 (T11-T15) | T11-15 | T1 + Q1 — 의료 SKU 2개 |
| 이재현 (Q1-Q5) | Q1-5 | T1 + Q7 (신뢰성 톤) — 의료 SKU 2개 |
| 최지혜 (Q6-Q10) | Q6-10 | T1 + Q1 — 의료·가글 1개씩 |

→ T1·Q1 차원에서 7명 독립 평가 = **Fleiss' kappa** 7-rater 계산 가능 (가장 robust)
→ 추가로 T6·Q7도 sub-pair kappa 측정
- 총 추가 cells: 7명 × 2 차원 × 2 SKU = **28 cells** (5분/인)
- 결과 jsonl: `data/processed/manual_tags_irr.jsonl`

## 6. 검수·수렴 — 일관성 보장

### Inter-rater agreement (Cohen's kappa)

이중 태깅 70 cells 결과로 차원별 kappa 계산:
- κ ≥ 0.8: 강한 일치 (그 차원은 신뢰 OK)
- 0.6 ≤ κ < 0.8: 적절한 일치 (수용 가능)
- κ < 0.6: 약한 일치 → 차원 정의 재논의 + Wayne 최종 판정

> 기록: `ml/results/manual_tag_irr.md` (분석은 데마 팀)

### Wayne 검수 우선순위

1. ★ 표시 cell (작업자 의심)
2. 이중태깅 불일치 cell
3. Lv4 도메인 위반 가능성 (M1·G3·M5·G3)
4. 자가모순 흔적 (예: T1 TABLE인데 Q1 SHALLOW)

### LLM 1차 분류 옵션 (시간 부족 시)

P0 차원 일부는 ChatGPT/Claude로 1차 분류 후 사람 검수:
- 비용: 13 SKU × $0.01 = ~$0.20
- 가능 차원: T1·T2·T11·M3·M5·M6·G1·G2·G5·G7·Q3·Q5·Q6·Q8 (객관적 키워드 검색 위주)
- 사람 검수 5초/cell × 13 = 약 1분/차원

LLM 1차 사용 시 절차:
1. `python3 scripts/llm_first_pass_tagging.py --vertical medical_device,gargle` (스크립트 있을 시)
2. `data/processed/manual_tags_llm_draft.jsonl` 생성
3. 사람이 빠르게 검수 + 수정
4. Step 4의 시간 50% 절감 가능

## 8. 데이터 활용 — 어디로 흐르나

수동 태깅 jsonl이 들어가는 곳:

### 데마 (Phase B2/B3)
- features.jsonl + manual_tags.jsonl 좌측 join (sku_id 키)
- B2 baseline 로지스틱: 페이지 피처 + 수동 태깅 → 멘션 확률
- B3 XGBoost + SHAP: 차원별 기여도 시각화 + 워터폴
- 컨설팅 액션: SHAP 음의 기여 차원이 곧 *개선 우선순위*

### 산공통 (본실험 보류 해제 시)
- F1·F2·F4·F5·F6 가설 검정에 페이지 실측 차원 사용
- *바디닥터의 F 수준이 어디인지* 실측 매핑 → 본실험 컨디션과 비교
- transferability 검증 (가상 페이지 효과 vs 실제 페이지 효과)
