# Wayne 진단 리포트 vs Claude 로드맵 통합

## 0. 결정적 발견 — Wayne 진단 리포트에 이미 풀 실험 결과 존재

| 지표 | 값 |
|------|-----|
| 측정 일자 | 2026-04-22 16:16 |
| 샘플 수 | **1,000건 AI 응답** |
| 멘션율 | 97.4% |
| **추천율(전체)** | **39.5%** |
| Top-1 추천율 | 62.1% |
| Top-3 노출율 | 66.0% |

→ 우리 본실험(5,000~7,500 호출)의 파일럿에 해당. 마스터 §9.3의 "14.5%" 수치 출처도 확정 (CAT 카테고리).

## 1. 쿼리 유형별 결과 — **이게 내가 제안한 7유형의 5개**

Wayne 진단은 이미 5유형을 가동 중 (용어 매칭):

| Wayne 분류 | 내 7유형 매칭 | 추천율 | Top-1 | 현상 |
|-----------|-------------|------|------|------|
| **BRD** (Brand-Named) | 브랜드명 | 53.0% | 84.0% | 중간 |
| **CAT** (Category General) | 일반 카테고리 | **14.5%** | 51.0% | 🔴 **최대 약점** |
| **SYM** (Symptom-Based) | 증상 해결 | 17.0% | 38.5% | 🔴 약점 |
| **CMP** (Comparative) | 비교 | 48.0% | 70.5% | 양호 |
| **COM** (Competitor-Named) | — (신규 유형) | 65.0% | 66.5% | 강점 |
| (미실행) | 가격·예산 | — | — | 본실험에 추가 |
| (미실행) | 유스케이스 | — | — | 본실험에 추가 |
| (미실행) | 구매 의도 (DECISION) | — | — | 본실험에 추가 |

**내 제안 수정**: 기존 7유형 + Wayne **COM** = **8유형 본실험** 권고. COM은 "경쟁사 이름이 포함된 쿼리"로 현재 가장 강한 영역(65%) → 왜 강한지 분석해서 약한 영역(CAT 14.5%)에 이식.

## 2. ★★ 닥터케이 실재 확인

Wayne 진단의 Head-to-Head 섹션:
> "닥터케이 저주파 EMS: 94.4% 승률, -46.0%p Top-1 격차 ← 압도적 우위"

→ **"닥터케이"는 "바디닥터K"의 약어 중복이 아니라 실재 브랜드**. 식약처 API 검색에서 못 잡힌 건 **품목명 기준 검색의 한계** (닥터케이가 다른 카테고리로 등록됐거나, 식약처 DB에 아예 없는 공산품 저주파자극기일 수도).

**액션**: 닥터케이 실제 URL/판매 채널 추적 → 본실험 경쟁군 포함 검토.

## 3. Wayne 진단에 있고 내 로드맵에 약한 것 (추가 반영 권고)

### 3.1 근거 유형 불균형 — "공식 < 커뮤니티"
> "상위 인용 도메인 7%만 bodydoctor.co.kr (통제 어려움)"

→ **External Evidence War의 구체 증거**. 93% 인용이 커뮤니티/블로그/리뷰. 공식 채널이 약함 → 정보 정확성·브랜드 통제력 리스크.

**로드맵 보강**: action_roadmap.md Stage 4 "External Evidence" 우선순위 상향. 공식 도메인 자체 콘텐츠 강화 (bodydoctor.co.kr 복구 + 풀 스키마) 먼저.

### 3.2 감성 분석 (긍정 71.7% / 중립 28.2% / 부정 0%)
> "부정 평가 없음 = 안정적, 하지만 중립 28.2%는 개선 여지"

→ 우리 Y 변수에 **감성** 없음. Y3 (근거 수치 기반도)와 별개 차원.

**Y 변수 추가 권고**: Y7 = 감성 분류 (긍정/중립/부정). API 응답 본문에 분류기 적용.

### 3.3 약점 프롬프트 개별 진단
> "BRD-050, BRD-086, BRD-093: 0% 추천 + 0% Top-1"

→ 개별 프롬프트 실패 사례 분석. 내 로드맵에 개별 프롬프트 분석 없음.

**분석 방법 추가**: 본실험 결과에서 **하위 10% 프롬프트**를 별도 케이스 스터디로 심층 분석.

### 3.4 반복 추이 추적
> "히스토리 1건만 존재 (변화 추이 향후 제공)"

→ 우리 프로젝트는 본실험 1회성. **정기 재측정 체계** 설계 안 됨.

**로드맵 보강**: Stage 6 "Measure"에 월간 주기적 재측정 명시.

## 4. 내 로드맵에 있고 Wayne 진단에 없는 것 (보완 가능)

### 4.1 Rufus SPN 5 facet 기반 페이지 콘텐츠 강화
Wayne 진단은 현재 추천율만 측정. **어떻게 개선**은 일반론 수준 ("CAT 강화").

**내 제안 유지**: Rufus SPN (Subjective Property + Activity + Target Audience) 명시 — CAT 14.5% → 목표 35% 도달의 구체 수단.

### 4.2 JSON-LD 풀 스키마 (MedicalDevice + offers + aggregateRating)
Wayne 진단은 JSON-LD 수준 측정 안 함.

**내 제안 유지**: action_roadmap.md Stage 3의 풀 스키마 코드.

### 4.3 Sixthshop 점수 체계
Wayne 진단은 "AI 응답" 기준만. 페이지 자체의 AI-readiness는 측정 안 함.

**내 제안 유지**: Sixthshop 4축(Schema/Content/Media/Commerce) 점수 자체 산출.

### 4.4 본실험 통제 실험(산공통)
Wayne 진단은 관찰 실측. **F 요인별 인과 효과**는 측정 불가.

**프로젝트 고유 가치**: 가상 페이지 L54 × F 변형으로 H1~H15 **인과** 검정.

## 5. 통합 제안 — Wayne 진단 + 프로젝트 = 1+1=3

| 영역 | Wayne 진단 (관찰) | 프로젝트 (통제 실험) | 통합 가치 |
|------|-----------------|----------------------|---------|
| 현 상태 측정 | ✅ 1,000건 실측 | — | **baseline** |
| 쿼리 유형별 성과 | ✅ BRD/CAT/SYM/CMP/COM | 쿼리 7유형 + 8유형(COM 포함) | **쿼리 설계 최적화** |
| 경쟁 비교 | ✅ 이지케이·닥터케이 | 본실험 N=6 통제 비교 | **인과적 차별점 추출** |
| F 요인 인과 | — | ✅ H1~H15 검정 | **개선 수단 정당화** |
| 외부 증거 분석 | ✅ 인용 도메인 7% | — | **로드맵 Stage 4 지시** |
| 감성 분석 | ✅ 긍정 71.7% | Y7 신규 추가 권고 | **안전성 측면** |
| 개별 프롬프트 실패 | ✅ BRD-050 등 | **하위 10% 케이스 스터디** 권고 | **핀포인트 개선** |
| 반복 추이 | ⏳ 2회차 대기 | 본실험은 1회성 | **월간 재측정 체계** |

→ Wayne 진단은 **"현재 어디가 문제인가"** (What + Where)
→ 프로젝트는 **"왜 문제인가 + 어떻게 고치면 효과가 얼마나 나오는가"** (Why + How much)
→ 로드맵은 **"실제 개선 실행"** (Execute)

셋을 묶으면 완성형. **Wayne 진단 리포트의 CAT 14.5% → 본실험의 F 수준 효과 크기 기반 After 예측 → 로드맵 실행 후 월간 재측정**.

## 6. 로드맵에 추가 반영할 항목

1. **공식 채널 인용 비율**을 Stage 4 핵심 지표로 상향 (현 7% → 목표 30%+)
2. Y7 감성 분류 신규
3. 본실험 후 **하위 10% 프롬프트 케이스 스터디** 섹션 추가
4. 월간 재측정 체계 (Wayne 진단 리포트 월 1회 재실행)
5. **COM 쿼리** (경쟁사 이름 포함) 쿼리 유형으로 추가 → 본실험 8유형
