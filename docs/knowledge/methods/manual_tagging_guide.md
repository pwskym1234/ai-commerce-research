> v1 (2026-04-25 오전): 페이지 구조 12개. **정성 차원 누락 지적 받음.**
> v2 (2026-04-25 보강): 페이지 구조 + 공통 정성 + 의료기기 특화 + 가글 특화 = **34개 차원**.

## 0. 분류 체계

| 그룹 | 항목 수 | 설명 |
|------|------|------|
| **A. 페이지 구조·형식** (T1~T12) | 12 | 자동 추출 외 사람 판단 필요한 페이지 디자인 차원 |
| **B. 공통 정성** (Q1~Q8) | 8 | 의료기기·가글 모두 적용. E-A-T·콘텐츠 깊이·신뢰 시그널 등 |

47 SKU 기준:
- 의료기기 ~30 SKU × (12 + 8 + 7) = 27개 차원 = **810 판정**
- 가글 ~17 SKU × (12 + 8 + 7) = 27개 차원 = **459 판정**
- **합계 1,269 판정**. 팀 3명 분배 시 1인당 ~420, 30초/판정 = 약 3.5시간/사람.

## 0.1 우선순위

각 차원에 P0/P1 표기. **P0만 먼저 태깅** 시 차원 약 절반 → 1.7시간/사람.

## 0.2 LLM 1차 분류 옵션 (반자동)

P1 차원의 일부는 ChatGPT/Claude에 페이지 텍스트 + 차원 질문 던져 1차 분류 후 사람 검수가 효율적. 방식:
1. 페이지 본문 텍스트 추출 (BeautifulSoup) → 3,000자 발췌
2. LLM에 "이 페이지에서 다음 항목들을 분류해주세요. {차원 정의}" 프롬프트
3. LLM 출력을 manual_tags_llm_draft.jsonl 저장
4. 사람이 5초/판정 검수만 수행

비용: 47 SKU × $0.01 = **~$0.50** (gpt-5.4-nano).

## B. 공통 정성 (Q1~Q8) — 신규

### Q1. 콘텐츠 깊이 (Content Depth) **P0**

> 페이지 본문이 단순 마케팅 카피인가, 진지한 사용 가이드/과학 설명까지 포함하는가?

| 값 | 기준 |
|----|-----|
| SHALLOW | 광고 카피 위주, 한 줄 셀링 포인트 반복 |
| MID | 스펙·사용법 정도 |
| DEEP | 과학 원리·임상 데이터·상세 사용 가이드·FAQ 풍부 |

### Q2. 위기 상황 가이드 (Safety Disclaimer) **P0**

> "이런 증상이면 의료진 상담" 같은 안내가 명시돼 있는가?

| 값 | 기준 |
|----|-----|
| EXPLICIT | 명확히 권고 ("X 증상이면 의사와 상담하세요") |
| IMPLICIT | "지속되면 병원 방문" 정도 흐릿한 언급 |
| NONE | 안내 없음 (의료기기는 표시기준 위반 가능성) |

### Q3. 학습 자료 (Educational Resources) **P1**

> 사용자가 별도로 볼 수 있는 PDF 매뉴얼·영상 가이드·운동 프로그램 등이 있는가?

| 값 | 기준 |
|----|-----|
| RICH | PDF + 영상 + 프로그램 등 다중 자산 |
| SOME | 1~2개 자산 |
| NONE | 페이지 내 텍스트 외 자산 없음 |

### Q4. 소셜 프루프 종류 (Social Proof Type) **P0**

> 어떤 사회적 증거를 사용하는가? 복수 선택 가능.

체크리스트 (해당 모두 체크):
- [ ] 누적 판매량 ("X만 개 판매")
- [ ] 후기 수 (구체 숫자)
- [ ] 평점 (별점 4.5+ 등)
- [ ] 미디어 노출 ("OO신문", "유튜브 OO")
- [ ] 전문가 인터뷰 (의사/약사/연구원)
- [ ] 인플루언서·연예인
- [ ] 글로벌 판매 ("17개국 4,000만 병" 같은)
- [ ] 수상·인증 (KGC인증, 대한민국 명품 등)

집계: 0~8.

### Q5. 환불·반품 정책 명확성 (Return Policy) **P1**

> 위생용품/의료기기 반품 제약을 명확히 표기하는가?

| 값 | 기준 |
|----|-----|
| CLEAR | 정책 표시 + 위생용품 반품 제약 명시 |
| PARTIAL | 일반 정책만, 위생용품 제약 누락 |
| MISSING | 환불·반품 정보 없음 |

### Q6. 고객 지원 표기 (Customer Support) **P1**

| 값 | 기준 |
|----|-----|
| RICH | 전화 + 채팅 + 이메일 + 운영 시간 |
| BASIC | 1~2 채널 |
| NONE | 지원 정보 없음 |

### Q7. 신뢰성 주장 톤 (Trust Claim Tone) **P0**

> "왜 우리를 믿어야 하는가" 의 핵심 어조는?

| 값 | 기준 |
|----|-----|
| AUTHORITY | 정부 인증·학회·전문가 인용 ("식약처 인증/대학병원 도입") |
| HERITAGE | 역사·전통 ("130년 역사", "원조 일본") |
| INNOVATION | 기술·특허 ("특허 출원", "AI 분석") |
| COMMUNITY | 사용자 다수 ("X만 명이 선택") |
| MIXED | 두 가지 이상 섞임 |
| WEAK | 명확한 신뢰 주장 없음 |

### Q8. 재구매 유도 메커니즘 (Retention Mechanism) **P1**

> 정기배송·멤버십·누적 할인 등 retention 장치가 있는가?

| 값 | 기준 |
|----|-----|
| YES | 정기배송/구독/누적 할인/멤버십 명시 |
| NO | 1회성 구매만 |

## C. 의료기기 특화 (M1~M7) — 신규

### M1. 효능 주장 톤 (Efficacy Claim) **P0**

> 의료기기 표시 가이드라인 준수 정도. "치료" 표현은 의료기기에만 허용. 공산품은 "운동기/관리기" 표현 강제.

| 값 | 기준 |
|----|-----|
| MEDICAL | "치료" "재활" "회복" 의학적 효능 명시 (의료기기 정당 사용 OR 공산품 위반 가능) |
| FUNCTIONAL | "운동" "강화" "관리" 기능적 표현만 |
| LIFESTYLE | "케어" "예방" "건강" 일반 표현 |
| AMBIGUOUS | 혼재 |

→ 의료기기인데 LIFESTYLE이면 페이지 약화. 공산품인데 MEDICAL이면 표시 위반.

### M2. 임상 근거 종류 (Clinical Evidence Type) **P0**

> 효능 주장의 뒷받침 근거.

| 값 | 기준 |
|----|-----|
| OWN_TRIAL | 자체 임상시험 결과 (KCT 번호·내부 데이터) |
| LITERATURE | 학술지 인용 (논문명·저널 명시) |
| EXPERT | 의사·약사 인터뷰 영상/글 |
| TESTIMONIAL | 사용자 후기로 의학적 효능 주장 (회색지대, 표시 위반 가능) |
| MARKETING | "특허 기술" "최첨단" 같은 마케팅 카피만 |
| NONE | 근거 자료 없음 |

### M3. 식약처 인증 등급 명시 (KFDA Class Display) **P0**

> 식약처 1/2/3등급, 제허 번호 명시 정도.

| 값 | 기준 |
|----|-----|
| EXPLICIT | "식약처 3등급 제허 X호" 명확히 표기 |
| PARTIAL | 등급 또는 번호 중 하나만 |
| KEYWORD_ONLY | "식약처 인증" 키워드만 |
| NONE | 표기 없음 |

### M4. 타겟 환자군 명시 (Target Patient Specificity) **P0**

> 어떤 사용자 그룹을 위한 제품인지 분기/명시 정도.

| 값 | 기준 |
|----|-----|
| SPECIFIC | "산후 6개월 이상 여성" "전립선 수술 후 회복기" 등 구체 |
| GENERAL | "골반저근 약한 분" "요실금 고민" 등 일반 |
| MISSING | 타겟 명시 없음 |

### M5. 부작용·주의사항 (Adverse Effects Disclosure) **P0**

> 의료기기 표시기준 ("심장 질환자 사용 금지" 등) 명시 여부.

| 값 | 기준 |
|----|-----|
| FULL | 별도 섹션 + 자세한 주의사항 |
| BRIEF | 1~2줄 짧은 경고 |
| MISSING | 부작용·금기 표기 없음 (의료기기 위반 가능) |

### M6. OEM/자체개발 표기 (Manufacturing Origin) **P1**

> 화이트라벨 OEM 제품인지 자체 개발인지. 바디닥터 OEM 의혹 관련 핵심.

| 값 | 기준 |
|----|-----|
| EXPLICIT_OWN | 자체 R&D 명시, 제조사 동일 |
| EXPLICIT_OEM | OEM/ODM 표기 또는 다른 제조사 명시 |
| AMBIGUOUS | 제조사 모호 ("국내 제조" 정도) |
| MISSING | 제조 정보 없음 |

### M7. 의료기관 도입 사례 (Clinical Adoption) **P1**

> 대학병원·재활의학과 등에 납품 표기.

| 값 | 기준 |
|----|-----|
| YES_NAMED | 기관명 명시 ("서울대병원 OO과 도입") |
| YES_GENERIC | "병원에서 사용중" 정도 |
| NO | 기관 도입 표기 없음 |

## D. 가글 특화 (G1~G7) — 신규

### G1. 의약외품 분류 명시 (Product Class Display) **P0**

> 식약처 분류 표기. 가글은 의약외품 vs 일반 화장품 vs 식품으로 분류 가능.

| 값 | 기준 |
|----|-----|
| EXPLICIT | "의약외품" 명확 표기 (또는 다른 분류 명시) |
| KEYWORD | "구강위생용품" 정도 흐릿 |
| MISSING | 분류 표기 없음 |

### G2. 유효 성분 함량 명시 (Active Ingredient Concentration) **P0**

> CPC(세틸피리디늄), 클로르헥시딘, 불소(ppm), 알코올 등 구체 함량 표기.

| 값 | 기준 |
|----|-----|
| FULL | 모든 유효 성분 + 함량 명시 |
| PARTIAL | 성분명만, 함량 미표기 |
| KEYWORD | "특수 성분" 마케팅 표현만 |
| NONE | 성분 정보 없음 |

### G3. 효능 주장 범위 (Efficacy Scope) **P0**

> 표시 가능 효능 한정. 가글 표시 가이드 준수 정도.

| 값 | 기준 |
|----|-----|
| LICENSED_ONLY | 식약처 허가 효능 ("구취 제거" "치아 청결") 만 |
| EXPANDED | "잇몸 질환 예방" "충치 예방" 까지 (의약외품 추가 효능) |
| OVERREACH | "잇몸 치료" "충치 치료" 의약품 영역 침범 (위반 가능) |
| AMBIGUOUS | 모호 |

### G4. 시각적 효능 시연 (Visual Efficacy Demo) **P0**

> 프로폴린스의 "찌꺼기 시각화" 같은 USP 시연 — 가글 차별화 핵심.

| 값 | 기준 |
|----|-----|
| STRONG | 색 변화·찌꺼기·미생물 등 시각적 직관 시연 |
| WEAK | 일반 사용 장면 |
| NONE | 시각 시연 없음 |

### G5. 알코올·민감 성분 표기 (Sensitive Ingredient Display) **P1**

| 값 | 기준 |
|----|-----|
| EXPLICIT | "무알코올" "저알코올" "X% 알코올" 명시 |
| MENTIONED | 성분표에만 |
| MISSING | 알코올·자극 성분 정보 없음 |

### G6. 사용 빈도·방법 권고 (Usage Recommendation) **P1**

> "1일 X회, 사용 후 30분 음식 섭취 금지" 등 구체 가이드.

| 값 | 기준 |
|----|-----|
| DETAILED | 빈도·시간·후속 행동 모두 안내 |
| BRIEF | "양치 후 사용" 정도 |
| MISSING | 사용법 안내 없음 |

### G7. 원산지·수입품 표기 (Origin Display) **P1**

> 일본 원조(프로폴린스) / 미국 (테라브레스) / 국산 등 명시. AI 추천 시 "국산" "일본 원조" 등 키워드가 영향.

| 값 | 기준 |
|----|-----|
| EXPLICIT | 원산지 + 제조국 명확 |
| BRAND_ONLY | 브랜드 국적 (Made in 표기 없음) |
| MISSING | 원산지 정보 없음 |

## 1. 태깅 절차

1. 태깅 스프레드시트(`data/processed/manual_tags_template.csv`) 열기
2. raw_path 컬럼의 HTML 파일 브라우저로 열기 (`open data/raw/...`)
3. 각 SKU의 vertical 컬럼 확인 → 의료기기면 A+B+C, 가글이면 A+B+D 차원만 답
4. 애매하면 'AMBIGUOUS' 또는 빈칸 → Wayne 일괄 판정
5. 한 SKU당 평균 1~2분 목표 (P0만 먼저면 30초)

## 2. 검수·수렴

- 팀원 3명이 같은 SKU 5~10개를 **이중 태깅** → inter-rater agreement (Cohen's kappa ≥ 0.6)
- 불일치 항목은 Wayne 최종 판정
- 결과 → `data/processed/manual_tags.jsonl`

## 3. 모델 통합

- features.jsonl + manual_tags.jsonl 좌측 join (sku_id key)
- categorical → one-hot (~50 컬럼 추가)
- 데마 B2/B3 모델에서 SHAP 결과 비교: "수동 태깅 차원이 자동 차원보다 기여도 큰가"
- 컨설팅 단계에선 정성 차원이 **개선 액션 직접 연결** — "M2 임상 근거가 NONE이면 → 임상 시험 발주 권고"

## 4. 산출물 위치

- 가이드 (이 파일): `docs/knowledge/methods/manual_tagging_guide.md`
- 템플릿 CSV: `data/processed/manual_tags_template.csv` (자동 생성 예정)
- 결과 jsonl: `data/processed/manual_tags.jsonl`
- LLM 1차 분류 초안 (옵션): `data/processed/manual_tags_llm_draft.jsonl`
- inter-rater 분석: `ml/results/manual_tag_irr.md`

## 5. 우선순위 P0 차원만 빠른 시작

P0만 처리하면 17개 차원, 47 SKU × 17 = 799 판정 / 30초 = 6.7시간 / 3명 = 2.2시간/사람.

P0 차원:
- A: T1, T2, T3, T7, T10
