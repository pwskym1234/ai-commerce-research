# 데마 데이터 신뢰도 평가

## 0. 결론 한 줄

**47 SKU는 ML 모델의 정확 예측에는 부족, SHAP 정성 패턴·EDA·진단 리포트에는 borderline 가능**. 정량 컨설팅 임팩트를 강화하려면 **80~150 SKU 추가 확보** 권장.

## 1. 통계학적 한계 — 권장 표본 vs 현재

| 분석 | Rule of thumb | 47 SKU 적용 |
|-----|--------------|-----------|
| 로지스틱 회귀 정확 예측 | feature 수 × 10. 30 feature → 300 | ❌ overfitting 매우 큼 |
| XGBoost 안정 예측 | 100+ | ❌ 분산 매우 큼 |
| SHAP 기여도 정성 트렌드 | 50+ | 🟡 borderline |
| EDA·기술 통계 | 30+ | ✅ 가능 |
| LOOCV 평균 성능 | n 자체로 가능 | ✅ (CI 매우 넓음) |
| Bootstrap CI | 1,000회 resampling | ✅ 가능 |

## 2. 47 SKU 분리 시 더 부족

- 두 버티컬 분리 (R10 룰): **의료기기 30 / 가글 17**
- Train/Test 분할 시 train 24 / test 6 (의료기기만 80:20)
- 가글은 N=17로 모델 학습 자체가 어려움
- 같은 SKU 채널별 row (바디닥터 5개) — 독립 표본 아님 → 사실상 18 브랜드

## 3. SKU 분포 현재 상태

```
의료기기 (30 SKU):
  bodydoctor      5 (gncosshop 채널 여러 페이지)
  furun           8 (자사몰 여러 변형)
  coway_therasol  4
  hools           3
  furenhealth     3
  applehip        2
  나머지 9 브랜드 각 1

가글 (17 SKU):
  listerine      10 (자사몰 여러 페이지, 0점 다수)
  garglin         2
  나머지 5 브랜드 각 1
```

→ listerine·furun·bodydoctor 외엔 1 SKU. variance 추정 거의 불가.

## 4. 보완 옵션 5개

### A. 추가 크롤로 SKU 100~150 확보 (권장)

추가 가능 풀:
- 식약처 OEM 풀 13개 중 미크롤 8개 (쉬엔비, 청우메디칼, 비엠씨, 유진플러스 등)
- 가글 신규 경쟁군 7개 (어썸쿨, 테라브레스, 유시몰, 콜게이트, 오랄비, 센소다인, 광동) — 경쟁군에는 추가됐지만 페이지 크롤 안 됨
- 동일 브랜드 다른 SKU (페로니언 다른 모델, 케겔매직 시리즈)

예상: +60~80 SKU → 총 110~130 SKU. 4~6시간 작업.

### B. B1-B 클로즈드셋 응답을 페이지 단위 Y로 사용 (이미 계획됨)

47 SKU × 24 쿼리 × 20 반복 = **22,560 페이지-쿼리 관측**

이건 **페이지 단위 Y** 라서:
- X: 페이지 피처 (47개)
- Y: 페이지가 추천 받았는가 (22,560 관측)
- 각 페이지는 24×20=480회 노출 → 페이지별 추천률 ±0.045 (95% CI)

→ ML 모델 학습용 표본 충분. 단 X 다양성은 47 페이지에 의존.

### C. Bootstrap + LOOCV (작은 표본 대응)

- LOOCV: 모든 메트릭에 47회 leave-one-out 평균
- Bootstrap 95% CI: "정확도 0.78 (95% CI 0.65~0.88)" 식으로 모든 결과에 CI 표기
- 결과 정직 표기 → 발표 신뢰도 ↑

### D. Feature 축소 (overfitting 완화)

- 30 feature → 10 (Lasso 자동)
- domain knowledge 기반 prior pruning
- 1차 선택 후 SHAP top 10 만 최종 모델

### E. 정량 예측 포기, 정성 SHAP만 보고

- "JSON-LD 추가 → 추천 +X%" 같은 정량 시뮬레이터 포기
- "JSON-LD 추가 → 추천 확률 증가 (방향성)" 정성 메시지만
- 컨설팅 임팩트는 약화되지만 정직

## 5. 추천 조합: B + A + C

1. **B**: 페이지 단위 Y로 22k 관측 활용 (B1-B 진행 시 자동)
2. **A**: 60~80 SKU 추가 크롤 — 의료기기 OEM 풀 + 가글 신규 경쟁군
3. **C**: 모든 결과에 Bootstrap CI 표기

이러면:
- ML 모델 학습 표본 충분
- 정량 예측 가능 (CI 동반)
- "47개 SKU의 22k 페이지-쿼리 관측" 으로 발표 정당화

## 6. Phase A+ 신설 — 추가 크롤 80~100 SKU

기존 로드맵 Phase A → Phase B 사이에 **A+ 추가 크롤** 단계 신설:

```
Phase A (완료) ─┬─ A+ 추가 크롤 80~100 SKU (4~6h)  ─┐
                ├─ 수동 태깅 P0 (병렬, 2.2h)       ├─> Phase B 강화
                └─ 외부 증거 NAVER 수집 (2분)      ─┘
```

Phase B는 이 보강 후 진행. 표본 110~130 SKU로 모델 안정성 ↑.

## 7. 정직한 발표 카피

> "본 연구는 47 SKU 페이지 × 24 쿼리 × 20 반복 = 22,560 페이지-쿼리 관측치를 사용했습니다. 페이지 다양성(N=47)은 ML 모델의 분산을 키우므로 모든 결과에 Bootstrap 95% CI를 동반 보고합니다. 향후 N≥150 확보로 신뢰구간을 1/√3 축소할 계획입니다."

이게 발표/컨설팅 모두에 정직 + 임팩트 있는 프레이밍.
