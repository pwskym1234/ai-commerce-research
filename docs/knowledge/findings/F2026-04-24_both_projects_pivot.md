## 1. 산공통 (통제 실험) — 변경 사항

### 1.1 가상 페이지 36 → **54개**

**이유**: L54가 H3(F1×F2)·H15(Q×F)·F4×F6 등 2차 교호작용 검정 가능. L36은 일부만.

**작업 영향**:
- 가상 페이지 제작 시간 약 50% 증가 (4시간 → 6시간)
- Jinja2 템플릿 한 번 완성 후 54 조합 자동 생성이라 선형 증가 아님
- HTML 검수 시간이 실제 병목

### 1.2 L54 직교배열 자동 생성

**방법**:
```python
# pyDOE2 또는 OATs 라이브러리
import pyDOE2
# L54는 일반 Taguchi 표로 제공 안 됨 — 3^6 fractional 설계 사용
design = pyDOE2.gsd([3]*6, reduction=13.5)  # 729/54 ≈ 13.5
# 또는 statsmodels · rpy2 + R DoE.base
```

**대안**: R `DoE.base::oa.design()` — 가장 신뢰성 높은 표준 표 생성.  
→ methods/orthogonal_array_L54.md 신규 작성 권고 (v1의 L36 버전 교체).

### 1.3 쿼리 8유형 × F 교호작용 (H15) 검정

v1 설계는 쿼리 요인이 **주효과** 취급. v2는 **Q × F 교호작용**까지 검정.

통계 모델:
```
logit(Y2a) = β0 + β_F1 × F1 + ... + β_F6 × F6
            + β_Q × Q
            + β_F1Q × (F1 × Q) + β_F6Q × (F6 × Q) + ...
            + ε
```

→ `statsmodels.formula.api.logit` 또는 `scikit-learn` + `PolynomialFeatures`로 교호작용 항 생성 후 회귀.

### 1.4 F1 "SPN 고정 주입" 제거 (실제 구현)

v1에서 혼란스러웠던 "F1 각 수준에 SPN 포함" 지시 **삭제**.

**변경**: F1은 순수 HTML 포맷만 조작. 스펙 정보는 세 수준 동일, SPN은 **조작 대상 아님**(기본 가상 페이지에 포함 안 함).  
H12(SPN 효과)는 **쿼리 유형 USE/SYM vs BRD/CAT의 F 효과 차이**로 검정 → H15 하위 가설.

### 1.5 새 가설 검정 방법

| 가설 | 검정 형태 |
|------|---------|
| H12 (SPN 맥락) | USE 쿼리 × F6(근거) 교호작용 항 유의성 |
| H14 (외부 증거) | **데마 영역** — 실크롤링 변수 + 관찰형 상관 |
| H15 (Q×F 교호) | L54 설계의 교호작용 항 ANOVA |

### 1.6 파일럿 — Wayne 진단 리포트 활용

기존 계획: 20~30조합 × 5회 파일럿 따로 실행.  
**신규**: Wayne 2026-04-22 진단의 **1,000건 실측이 사실상 파일럿**. 본실험 직접 돌입 가능.  
→ 파일럿 비용 $14 절감.  
→ Wayne 진단의 `추천율 CAT 14.5% / BRD 53%`를 본실험 **사전 신호**로 활용해 효과 크기 추정.

### 1.7 Y 변수 측정 로직

| Y | 측정 방식 | 도구 |
|---|---------|------|
| Y1 파싱 정확도 | 응답에서 "99단계/X Hz" 추출, 원본 스펙과 일치 | 정규식 + fuzzy match |
| Y2a 멘션 | 브랜드명 NER | 단순 `"바디닥터" in response` (오탐 대비 경쟁사명 배제) |
| Y3 근거 기반도 | 응답 본문을 분류기로 "스펙 vs 감정" 구분 | 분류 프롬프트 (gpt-4o-mini) 또는 키워드 룰 |
| Y4 안전성 회피 | "전문가와 상담", "의사 진료" 키워드 감지 | 키워드 룰 + 비율 |
| Y5 상대 순위 | 응답 내 브랜드 나열 순서 | 순서 파싱 |
| Y6 Citation | URL 추출 후 bodydoctor.co.kr/gncosshop.com 매칭 | 도메인 파싱 |
| Y7 감성 | 긍정/중립/부정 분류 | gpt-4o-mini 분류 프롬프트 또는 KR-BERT |

## 2. 데마 (예측·관찰) — 변경 사항

### 2.1 관찰 변수 대폭 확장

기존 피처(`extract_features.py`)에 신규 추가:

| 신규 변수 | 출처 |
|---------|------|
| **Sixthshop 4축 점수** (A/B/C/D + 총점) | `sixthshop_score.py` 신규 스크립트 |
| **External Evidence 풍부도** (리뷰 수·외부 언급 수) | 경쟁사별 수집 |
| **Citation Source 분포** (공식 / 커뮤니티 / 리뷰) | Wayne 진단 리포트 방식 |
| **감성 분포** (Y7과 동일 기준) | 응답 분류 |

### 2.2 H14 (외부 증거) 관찰형 검정

**모델**:
```
Y2a = f(Sixthshop 점수, 리뷰 수, 외부 인용 수, 가격, 카테고리, ...)
```

**기법**: 
- Baseline: 로지스틱 회귀
- Main: XGBoost + SHAP
- **SHAP로 "외부 증거" vs "페이지 구조"의 상대 기여도 비교**

### 2.3 산공통 결과 + 데마 데이터 결합 분석

**통합 스토리**:
1. 산공통에서 F 요인의 **인과 효과 크기** 추정 (예: F5=풀 → Y2a +15%)
2. 데마에서 실제 SKU가 그 F 수준에 있는지 측정
3. 둘을 결합: "바디닥터가 F5=없음 → F5=풀 로 가면 Y2a +15%" 구체 예측

이게 **컨설팅 Before/After 시뮬레이터의 핵심 로직**.

### 2.4 쿼리 유형별 성과 + F 매칭 (신규 분석)

Wayne 진단에서 CAT 14.5% vs BRD 53% 격차가 큼.  
→ 데마에서 각 쿼리 유형별로 "어느 F 수준이 가장 효과 큰지" 분리 학습.  
→ 쿼리 유형별 **개선 우선순위** 도출 (예: CAT 쿼리엔 F2 강화가 가장 효과적).

### 2.5 새로운 EDA 축

- 경쟁사 Sixthshop 점수 분포 (v2 전 실측 완료: [sixthshop_scores.jsonl](../../../data/processed/sixthshop_scores.jsonl))
- 자사몰 vs 이커머스 플랫폼 점수 비교 (프로폴린스 36 vs 2080 SSG 82)
- 채널별 AI 노출 분포 (같은 제품의 채널별 Sixthshop 점수 차이)

## 3. 공통 분석 — 버티컬 비교

### 3.1 의료기기 vs 가글 추천 드라이버 차이 (마스터 §5.4)

v2 신규 변수로 비교 깊이 증가:

| 드라이버 | 의료기기 (예상 SHAP rank) | 가글 (예상 SHAP rank) |
|---------|-----------------------|---------------------|
| Sixthshop Schema 점수 | 1위 | 2위 |
| 인증 정보 풀 명시 (F5) | 1~2위 | 하위 (na) |
| 임상 인용 (F6) | 2위 | 하위 |
| **리뷰 수 (external)** | **중위 (신규)** | **1위 (신규)** |
| 가격 | 중위 | 중위 |
| Y7 감성 분포 | **중위 (신규)** | **중위 (신규)** |

### 3.2 Wayne 진단 리포트 상호 검증

본실험 결과 + Wayne 진단 결과를 **교차 검증**:
- Wayne 진단: CAT 추천율 14.5%
- 본실험 예측: "CAT 쿼리 + F5=없음" 수준의 예측 추천율
- 둘이 근사하면 본실험 외적 타당성 입증
