## 0. 작업 한 줄 요약

## 4. 진행 절차 — 단계별

### Step 1 — 환경 준비 (10분, 모든 팀원)

- [ ] 본인 분담 차원 확인 (위 §1 표)
- [ ] [manual_tagging_guide.md](manual_tagging_guide.md) 본인 차원 섹션 정독 (15분 이내)
- [ ] CSV 템플릿 열기: `data/processed/manual_tags_template.csv` (Excel/Google Sheets)
  - 행: 13 SKU
  - 열: 39 차원 — 본인 담당 컬럼만 채움. 나머지는 `N/A` 또는 빈칸
- [ ] 슬랙 #aieo-수동태깅 채널 join (질문·이중태깅 협업)

### Step 2 — 페이지 raw HTML fetch (팀원 1·2만)

**팀원 1** (의료 9 페이지):
```bash
# 자사몰 fetch — curl_cffi (Cloudflare 우회)
python3 crawler/scripts/fetch_competitor_pages.py --vertical medical_device --brands bodydoctor,easyk,coway_therasol,ceragem_innerfit,elvie,perifit,kegel8,kgoal,peroneun
# → data/raw/medical_device/<brand>/2026-04-30/index.html 저장
```

**팀원 2** (가글 4 페이지):
```bash
python3 crawler/scripts/fetch_competitor_pages.py --vertical gargle --brands propolinse,listerine,garglin,perio
```

> 스크립트가 없으면 데마 팀에 요청 (curl_cffi Chrome impersonate 메모리 룰 준수). raw HTML이 안 깨끗하면 Playwright fallback.

### Step 3 — 첫 SKU 1개 연습 (10분, 모든 팀원)

- [ ] 자기 분담 차원 중 *첫 SKU 1개*만 시범 태깅
- [ ] 슬랙에 결과 공유 → Wayne 또는 도메인 specialist 한 번 확인
- [ ] 막히는 차원은 빈칸으로 두고 슬랙 질문

### Step 4 — 본격 태깅 (60~70분, 모든 팀원)

각자 분담 차원 × 13(또는 9·4) SKU 채워 넣기.

**진행 룰**:
- 한 SKU당 30~90초 안에 결정 (난이도별 §2)
- 5초 안에 결정 안 되면 **빈칸 + 슬랙 질문**
- *시각 인상* 차원(T6 톤 등)은 페이지 첫 인상 그대로 — 너무 분석하지 말 것
- 의심 가는 케이스는 cell에 `★` 표시 (검수 우선순위)

### Step 5 — 자체 검토 (10분, 모든 팀원)

- [ ] 본인 채운 cell 일관성 체크 (예: 같은 SKU에서 T1 TABLE인데 T4 LOW면 자가모순?)
- [ ] 빈칸·★ cell 정리 → 슬랙에 질문 마지막 정리

### Step 6 — 이중 태깅 (15분, 팀원 7 + 1명 paired)

> Cohen's kappa ≥ 0.6 목표 (manual_tagging_guide.md §F)

각 P0 차원에 대해 *2 SKU 무작위 추출* → 분담자 외 1명이 같은 차원 독립 태깅 → 일치도 측정.
- 예: 팀원 3 (T1-T5) 가 채운 13 SKU 중 2개 → 팀원 7이 같은 5 차원 독립 태깅
- 7쌍 × 2 SKU × 5 차원 = 70 cells 추가 (5~10분/인)
- 결과 jsonl: `data/processed/manual_tags_irr.jsonl`

### Step 7 — Wayne 검수 (Wayne 30분)

- [ ] 모든 ★ 표시 cell 검토
- [ ] 이중태깅 *불일치* cell 최종 판정
- [ ] Lv4 도메인 케이스 (M1 효능 톤·G3 효능 범위 등) 위반 가능성 검토 → 결과 적층

### Step 8 — 최종 jsonl 머지 (1명 담당)

```bash
python3 ml/scripts/merge_manual_tags.py \
  --csv data/processed/manual_tags_template_filled.csv \
  --output data/processed/manual_tags.jsonl
```

> 출력: `data/processed/manual_tags.jsonl` (sku_id 키, 데마 B2/B3 + 산공통 본실험 가설 검정에 join 입력)

## 6. 검수·수렴 — 일관성 보장

### Inter-rater agreement (Cohen's kappa)

이중 태깅 70 cells 결과로 차원별 kappa 계산:
- κ ≥ 0.8: 강한 일치 (그 차원은 신뢰 OK)
- 0.6 ≤ κ < 0.8: 적절한 일치 (수용 가능)
- κ < 0.6: 약한 일치 → 차원 정의 재논의 + Wayne 최종 판정

> 기록: `ml/results/manual_tag_irr.md` (분석은 데마 팀)

### Wayne 검수 우선순위

1. ★ 표시 cell (작업자 의심)
2. 이중태깅 불일치 cell
3. Lv4 도메인 위반 가능성 (M1·G3·M5·G3)
4. 자가모순 흔적 (예: T1 TABLE인데 Q1 SHALLOW)

### LLM 1차 분류 옵션 (시간 부족 시)

P0 차원 일부는 ChatGPT/Claude로 1차 분류 후 사람 검수:
- 비용: 13 SKU × $0.01 = ~$0.20
- 가능 차원: T1·T2·T11·M3·M5·M6·G1·G2·G5·G7·Q3·Q5·Q6·Q8 (객관적 키워드 검색 위주)
- 사람 검수 5초/cell × 13 = 약 1분/차원

LLM 1차 사용 시 절차:
1. `python3 scripts/llm_first_pass_tagging.py --vertical medical_device,gargle` (스크립트 있을 시)
2. `data/processed/manual_tags_llm_draft.jsonl` 생성
3. 사람이 빠르게 검수 + 수정
4. Step 4의 시간 50% 절감 가능

## 8. 데이터 활용 — 어디로 흐르나

수동 태깅 jsonl이 들어가는 곳:

### 데마 (Phase B2/B3)
- features.jsonl + manual_tags.jsonl 좌측 join (sku_id 키)
- B2 baseline 로지스틱: 페이지 피처 + 수동 태깅 → 멘션 확률
- B3 XGBoost + SHAP: 차원별 기여도 시각화 + 워터폴
- 컨설팅 액션: SHAP 음의 기여 차원이 곧 *개선 우선순위*

### 산공통 (본실험 보류 해제 시)
- F1·F2·F4·F5·F6 가설 검정에 페이지 실측 차원 사용
- *바디닥터의 F 수준이 어디인지* 실측 매핑 → 본실험 컨디션과 비교
- transferability 검증 (가상 페이지 효과 vs 실제 페이지 효과)

### 컨설팅 (Phase C)
- 바디닥터 자사몰 진단 표 (F1~F6 + Sixthshop 점수 + Q1·Q9 외부 권위 등)
- Before/After 시뮬레이터 입력값
- 우선순위 액션 리스트의 정량 근거

## 10. FAQ

**Q: 5초 안에 결정 안 되면?**  
A: 빈칸 + 슬랙 질문. 끌고 가지 말 것.

**Q: 자사몰 raw HTML이 깨졌거나 로그인 벽이면?**  
A: 슬랙 보고 — 데마 팀에서 Playwright stealth 또는 다른 채널 시도. 그 SKU는 그동안 빈칸.

**Q: M·G 도메인 차원인데 본인이 그 SKU 안 봐도 되나?**  
A: 의료 SKU의 G 차원은 N/A 자동 (가글 SKU의 M도 마찬가지). 본인 vertical SKU만 입력.

**Q: 페르소나가 차원에 영향 주나?**  
A: 수동 태깅은 *페이지 자체의 객관적 특성* — 페르소나 무관. 페르소나는 본실험 *쿼리*에 적용 (queries_*_페르소나반영.yaml).

**Q: 이중 태깅 결과가 자기랑 다르면 누가 맞나?**  
A: 1차로 두 사람이 슬랙에서 토론 → 5분 안에 합의 안 되면 Wayne 검수. 자기 답 *수정 금지* (원본 보존, 일치도 측정용).

**Q: 본 작업이 본실험 재실행과 어떻게 연결?**  
A: 두 작업 *병렬*. 본실험 재실행은 LLM 호출(Y 변수 확보), 수동 태깅은 페이지 피처(X 변수 확보). 둘이 머지되어야 데마 모델 학습 가능.
