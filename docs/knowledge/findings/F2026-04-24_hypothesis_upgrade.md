# 업계 실태 기반 F 요인·가설 재설계

## 0. 고도화 원칙

Wayne 리서치 2건을 종합한 **이 시장의 실제 동작 로직**을 프로젝트 F 요인에 반영. 핵심 관찰:

1. **현재 F1~F6은 "페이지 기술 구조"에 쏠림** — Rufus/Azoma/Alhena가 실제 평가하는 "의미적 맥락"(Rufus SPN 5facet)은 빠짐
2. **Y2 이진(추천 O/X)은 너무 거칠음** — 업계는 "rendering"을 함께 봄 (단순 멘션 vs rich commerce card)
3. **"외부 증거 생태계" 차원 누락** — 리뷰·제3자 기사·Reddit·커뮤니티 신호
4. **쿼리셋이 4유형뿐** — 업계는 7개 유형 (브랜드/카테고리/증상해결/비교/가격/유스케이스/구매의도)
5. **MedicalDevice JSON-LD만으로 부족** — ChatGPT Shopping card 트리거는 별도 필드 (review_count, average_rating, availability, offers 등) 완성도에 의존

→ 본실험 L36 36조합 제약 유지 위해 **F는 6개 그대로, 각 수준 정의를 정밀화**. 새 차원은 **관찰 변수(데마) + 신규 가설 H12~H15** 로 추가.

## 1. 기존 F1~F6 수준 정의 정밀화

### F1 HTML 포맷 (기존: TABLE/BULLET/PARAGRAPH)

**정밀화**: 포맷 자체 + **의미적 밀도** 함께. 3수준 유지하되:

| 수준 | 정밀 정의 |
|------|---------|
| TABLE | 스펙(주파수/단계/시간/모드) **+ Rufus SPN 일부(Activity Suitability: "산후 회복용", Target Audience: "출산 여성")** 를 `<table>` 구조로. row ≥ 6. |
| BULLET | 동일 정보를 `<ul><li>`로. row ≥ 8. |
| PARAGRAPH | 동일 정보를 `<p>` 산문으로 풀어서. 문장 ≥ 6. |

→ 정보 동등성 유지가 R7 (요인 외 변동 통제). Rufus SPN은 "있음" 고정 — 이건 F1과 직교.

### F2 JSON-LD (기존: MedicalDevice / Product / 없음)

**정밀화**: "Shopping card-ready" 여부로 재정의.

| 수준 | 정밀 정의 | 필드 수 |
|------|---------|------|
| MedicalDevice-Full | `@type=MedicalDevice` + brand/description/manufacturer/model/identifier + **AggregateRating + offers (availability, price, priceValidUntil) + review_count** | 11+ |
| Product-Minimal | `@type=Product` + name/brand/price/image만 | 4~5 |
| 없음 | JSON-LD 태그 없음 | 0 |

→ 수준 차이의 크기가 "Shopping card 트리거 가능성"에 대응.

### F3 수치 구체성 (기존 2수준 → 3수준 확장, methods/orthogonal_array_L36.md에 이미 반영)

**정밀화**: Rufus가 읽는 **제품 속성 필드 완성도** 관점 추가.

| 수준 | 정밀 정의 | 예시 |
|------|---------|------|
| 정확 | 정량 수치 + 단위 + 검증 가능한 스펙 | "99단계 강도, 50Hz 저주파, 1회 15분" |
| 부분 | 일부 수치만 | "다단계 강도, 자동 프로그램, 짧은 시간 사용" |
| 모호 | 마케팅 표현만 | "강력한 EMS, 다양한 강도, 편리한 자동 모드" |

### F4 인증 정보 위치 (기존 유지: 상단/하단/없음)

### F5 인증 명시 수준 — 재정의 (기존 3수준 → 더 정밀)

| 수준 | 정밀 정의 |
|------|---------|
| 풀 | "식약처 **3등급** 의료기기 — 비이식형 요실금 신경근 전기자극장치 (허가번호 **제허 XX-329 호**) / FDA **Class 1·2** 등록" |
| 부분 | "식약처 인증 의료기기" 또는 "FDA 등록" 단독 |
| 없음 | 인증 관련 어떤 텍스트도 없음 |

### F6 근거 유형 — 재정의 (기존 3수준 유지, 의미 확장)

| 수준 | 정밀 정의 |
|------|---------|
| 임상 | **제3자 기관** 임상 데이터 인용 (예: "OO대학병원 RCT, 4주 32% 향상, n=120, p<0.01") |
| 후기 | 사용자 후기 + 셀럽 추천 (예: "30대 여성 200건 후기", 김상중 같은 모델) |
| 없음 | 근거 영역 자체 없음 |

## 2. 신규 가설 H12~H15 (사전 등록)

### H12 — SPN(맥락 서술) 명시가 추천 ↑ ★ Rufus 논문 기반

**내용**: 제품 설명에 **Subjective Property + Activity Suitability + Target Audience**를 명시하면 AI 추천 확률이 증가한다.

**메커니즘**: Amazon Rufus 논문(2024)이 드러낸 5 facet 구조. ChatGPT Shopping·Perplexity도 유사 패턴으로 제품 의도 매칭.

**운용**: F1~F6와 독립된 **보조 변수(SPN_present: yes/no)**로 각 페이지에 고정 주입 후 효과 측정. L36 바깥의 파일럿 조건으로 2종 추가 (36×2 = 72 페이지? 비용 +2배 → 재검토 필요) 또는 쿼리 레벨에서 통제.

**간소화 권고**: L36 유지 + **쿼리셋에 SPN 매칭 유형 추가**. 쿼리가 SPN을 직접 호출할 때 추천율 차이 측정.

### H13 — Shopping Card Rendering 전환율 (Y2 이중 측정)

**내용**: Y2(추천)를 **(a) 텍스트 멘션** vs **(b) rich card rendering (가격/리뷰/이미지 포함)** 2수준으로 분리 측정 시, F2 수준에 따른 효과 크기 다르다.

**메커니즘**: Alhena 리서치 — "rendering"이 실제 매출 전환에 직결. 단순 멘션은 인지에만, rich card는 구매에 영향.

**운용**: Y2를 Y2a(멘션), Y2b(card rendering) 2변수로 동시 측정. ChatGPT Shopping, Perplexity Pro 응답의 카드 구조 파싱 필요.

### H14 — 외부 증거 풍부도가 추천 ↑ (관찰형)

**내용**: 브랜드별 외부 증거 지수(리뷰 수 + 제3자 기사 수 + Reddit·카페 언급 수)가 높을수록 AI 추천 확률이 증가한다.

**운용**: 
- 산공통 통제 실험의 F로는 **넣지 않음** (외부 증거는 가상 페이지 조작 불가)
- 데마 실제 크롤링 데이터에서 관찰 변수로 측정
- "바디닥터 리뷰 수 / 프로폴린스 리뷰 수" 같은 브랜드 수준 변수로 사용

**메커니즘**: 리서치 §10.4 "제품 단위 사업은 결국 external evidence war". Profound의 Citation Categories 연구.

### H15 — 쿼리 유형 × F 교호작용

**내용**: F 효과 크기가 쿼리 유형에 따라 다르다. 특히 **비교형 쿼리**에서 F2(JSON-LD), F5(인증)의 효과가 정량형 쿼리보다 크다.

**메커니즘**: 비교 쿼리는 여러 브랜드 동시 평가 → 구조화 데이터가 있는 쪽이 "정보 풍부도" 우위.

**운용**: L36 설계에 쿼리 4유형 × 조합 36을 직교 배치. 교호작용 F × Q 검정.

## 3. 쿼리셋(Q1) 재설계

기존 4유형 → **7유형** (업계 리서치 §8.1 반영):

| 유형 | 설명 | 예시 |
|------|------|------|
| 브랜드명 | 브랜드 직접 질문 | "바디닥터 요실금치료기 어때요?" |
| 일반 카테고리 | 카테고리 추천 요청 | "케겔운동기 추천해줘" |
| **증상 해결** | 문제 중심 | "요실금 예방에 좋은 가정용 기기" |
| **비교** | 2~3종 비교 | "바디닥터 vs 이지케이 뭐가 나아?" |
| **가격·예산** | 예산 기반 | "100만원 이하 요실금치료기" |
| **유스케이스** | 상황 특정 | "출산 후 6개월 회복용" |
| **구매 의도** | DECISION | "하나만 골라줘" |

**본실험 설계 업데이트**: L36 × 7쿼리 × 20반복 × 1 LLM = **5,040 호출** (~$50, 기존 2,880 대비 +75%).

비용 관리 옵션:
- 쿼리를 **코어 4 + 확장 3** 으로 분할, 코어는 20회 / 확장은 10회 → **4,320호출**로 억제
- 파일럿에서 각 쿼리 유형별 변동성 보고 본실험 범위 조정

## 4. Y변수 재설계 (Y1~Y6)

기존 Y1~Y4에서 업계 rendering 구분 반영.

| 변수 | 이름 | 정의 | 측정 방법 |
|------|------|------|---------|
| **Y1** | 파싱 정확도 | AI가 스펙 수치를 정확히 읽음 | 응답에서 "99단계/50Hz" 추출 일치도 |
| **Y2a** | 멘션 선택률 | 우리 제품이 텍스트에 멘션됐는가 | 정규식 + brand NER |
| **Y2b** | Rich Card 렌더링 | ChatGPT Shopping card로 렌더링됐는가 | 응답 구조 파싱 (있으면 "카드 객체" 감지) |
| **Y3** | 근거 수치 기반도 | 추천 이유가 스펙/인증 기반 vs 감정 | 응답 텍스트 분류 (키워드 + 분류기) |
| **Y4** | 안전성 회피 반응 | "전문가와 상담하세요"로 회피 | 키워드 감지 + 회피율 계산 |
| **Y5** | 상대 순위 | 경쟁군 N=6 내 우리 위치 | 1~6위 → 순위 평균 |
| **Y6** | Citation 포함 | 응답이 우리 공식 URL 인용 | URL 추출 + 도메인 매칭 |

→ Y2 이중 정의(이진 + 연속) 유지하되, "이진"을 Y2a, "연속"을 "N회 반복 중 선택 빈도" 로 나누고 추가로 Y2b(rendering) 독립 측정.

## 6. 비용/복잡도 영향

| 변경 | 본실험 호출 수 | 비용 증가 |
|------|-------------|---------|
| 기존 설계 | 36 × 4 × 20 × 1 = 2,880 | $29 |
| 쿼리 7유형 (동일 반복) | 36 × 7 × 20 × 1 = 5,040 | $50 |
| 쿼리 코어/확장 분할 | 36 × (4×20 + 3×10) = 3,960 | $40 |
| Y2b 측정 추가 | 동일 호출 수 (응답 파싱만 추가) | $0 |
| H12 SPN 고정 주입 | 설계만 변경, 호출 수 동일 | $0 |

**권고**: 쿼리 코어 4유형(기존) + 확장 3유형(10회) = 3,960 호출, $40. 비용 +40% 로 허용 범위.

## 7. 학술적 강화 효과

발표에서 얻는 임팩트:
- **H12 (Rufus SPN)**: Amazon 자체 논문 인용으로 "업계 표준을 우리 실험에 적용" 스토리
- **Y2a/Y2b 분리**: "단순 추천이 아닌 rich card rendering을 구분 측정"이라는 측정 정밀성 어필
- **H14 외부 증거**: 데마 결과에서 "페이지 개선만으론 부족, external evidence가 X% 설명력"
- **쿼리 7유형**: 산업 표준 쿼리 분류 적용으로 일반화 강화
