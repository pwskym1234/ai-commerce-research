## 4. 진행 절차 — 단계별

### Step 1 — Google Sheets 셋업 (Wayne, 10분)

본 작업은 **Google Sheets에서 7명 동시 작업**. Wayne이 셋업:

1. **빈 Google Sheet 생성**: 제목 *"AiEO 수동 태깅 — 2026-04-30"*
2. **CSV import**: 메뉴 `파일 > 가져오기 > 업로드 > [data/processed/manual_tags_template.csv](../../../data/processed/manual_tags_template.csv)` → "현재 시트 바꾸기"
3. **편집 권한 부여**: 7명(박지윤/이소현/이경민/박재후/최지혜/이재현/방우식)에게 *편집자* 권한 + 슬랙 #aieo-수동태깅 채널에 Sheet URL 공유
4. **컬럼 분담 가이드**: 시트 1행에 *각 차원 담당자 이름* 코멘트 (예: T1 컬럼에 "이경민" 코멘트). 또는 §1 분배 표 별도 시트로 첨부.
5. **N/A 자동 채움 확인**: CSV에서 의료 SKU의 G* / 가글 SKU의 M* 칸은 이미 `N/A` — 작업자가 건드리지 않음

> Google Sheets는 *동시 편집*되니 7명 한꺼번에 다른 컬럼 채움 가능. 단 *같은 cell* 동시 편집 충돌 회피 (분담 표대로 본인 컬럼만).

### Step 2 — URL 사전 검증 (박지윤·이소현, 30분)

각자 vertical SKU URL을 *브라우저로 한 번씩 열어* 200 OK + 의미 있는 페이지인지 확인. 차단·DNS 실패 시 슬랙 보고 → fallback 결정.

**박지윤** (의료 9 URL):
- bodydoctor.co.kr / easyk.kr / coway.com prdno=1360 / ceragemmall / elvie.com / perifit.co / kegel8.co.uk / kgoal.com / st365.co.kr 페로니언

**이소현** (가글 4 URL):
- propolinse.co.kr / listerine.kr / dmall.co.kr garglin / lghnh.com perio

> 차단된 페이지는 *비공식 fallback* (예: 네이버 스마트스토어 제품 페이지) 검토 + Wayne 컨펌. 또는 그 SKU 빼고 진행.

### Step 3 — 환경 준비 + 첫 SKU 연습 (10분, 모든 팀원)

- [ ] 슬랙 #aieo-수동태깅에서 Google Sheets URL 받기
- [ ] [manual_tagging_guide.md](manual_tagging_guide.md) 본인 차원 섹션 정독 (15분 이내)
- [ ] 본인 담당 컬럼 확인 (Sheet 1행 코멘트 또는 §1 분배 표)
- [ ] 첫 SKU 1개의 본인 차원 시범 태깅 → 슬랙 공유

### Step 4 — 본격 태깅 (Google Sheets에서 동시, 35~70분)

각자 *Google Sheets에서* 본인 분담 컬럼 × 13(또는 9·4) SKU 채워 넣기.

작업 룰:
1. 한 row 선택 → `url` 컬럼의 자사몰 링크를 **새 탭에서 열기** (Ctrl/Cmd + 클릭)
2. 페이지를 *위에서 아래로* 빠르게 스크롤하며 본인 차원 판정 (한 SKU 30~90초)
3. Google Sheets로 돌아와서 본인 컬럼에 값 입력
4. 의심 가는 케이스는 cell *노란색* + 코멘트로 "검수 필요" 표시
5. 5초 안에 결정 안 되면 빈칸 + 슬랙 질문

### Step 5 — 자체 검토 (10분)

- [ ] 본인 채운 cell 일관성 체크 (예: 같은 SKU에서 T1 TABLE인데 T4 LOW면 자가모순?)
- [ ] 빈칸·노란색 cell 정리 → 슬랙에 마지막 질문

### Step 6 — 이중 태깅 (15분, 7명 모두)

> Cohen's / Fleiss' kappa ≥ 0.6 목표. 일치도 측정용.

**핵심 차원 cross-rater 검정** — 모든 팀원이 *자기 분담 차원 외에* 핵심 P0 차원 2개의 같은 SKU 2개를 독립 태깅 (Sheet의 별도 시트 *IRR* 에 입력):

| 팀원 | 본인 분담 차원 | 추가 cross 차원 (이중 태깅용) |
|:-:|---|---|
| 박지윤 (M1-M7) | M | T1 (주도 포맷) + Q1 (콘텐츠 깊이) — 의료 SKU 2개 |
| 이소현 (G1-G7+) | G | T1 + Q1 — 가글 SKU 2개 |
| 이경민 (T1-T5) | T1-5 | T6 (메시지 톤) + Q1 — 의료 SKU 2개 |
| 박재후 (T6-T10) | T6-10 | T1 + Q1 — 의료 SKU 2개 |
| 방우식 (T11-T15) | T11-15 | T1 + Q1 — 의료 SKU 2개 |
| 이재현 (Q1-Q5) | Q1-5 | T1 + Q7 (신뢰성 톤) — 의료 SKU 2개 |
| 최지혜 (Q6-Q10) | Q6-10 | T1 + Q1 — 의료·가글 1개씩 |

→ T1·Q1 차원에서 7명 독립 평가 = **Fleiss' kappa** 7-rater 계산 가능 (가장 robust)
→ 추가로 T6·Q7도 sub-pair kappa 측정
- 총 추가 cells: 7명 × 2 차원 × 2 SKU = **28 cells** (5분/인)

### Step 7 — Wayne 검수 (Google Sheets 안에서, 30분)

- 노란색 코멘트 표시된 cell 검토
- 이중태깅 불일치 cell 최종 판정
- Lv4 도메인 위반 가능성 (M1·G3·M5) 검토
- 자가모순 흔적 (예: T1 TABLE인데 Q1 SHALLOW) 검토

### Step 8 — Sheet → CSV 다운로드 + jsonl 머지 (Wayne, 10분)

1. Google Sheets에서 `파일 > 다운로드 > 쉼표로 구분된 값(.csv)` 선택
2. 다운로드된 파일을 `data/processed/manual_tags_filled.csv` 로 저장 (repo에 push)
3. jsonl 머지 (Step 6 IRR 시트도 별도 다운로드 → manual_tags_irr.csv → jsonl):
   ```bash
   python3 ml/scripts/merge_manual_tags.py \
     --csv data/processed/manual_tags_filled.csv \
     --output data/processed/manual_tags.jsonl
   ```

> 출력: `data/processed/manual_tags.jsonl` (sku_id 키, 데마 B2/B3 + 산공통 본실험 가설 검정에 join 입력)
> Google Sheet은 *작업 매체*, repo의 csv·jsonl이 *최종 산출물*

## 6. 검수·수렴 — 일관성 보장

### Inter-rater agreement (Fleiss' kappa)

이중 태깅 28 cells 결과로 차원별 kappa 계산 (T1·Q1 7-rater, T6·Q7 sub):
- κ ≥ 0.8: 강한 일치 (그 차원은 신뢰 OK)
- 0.6 ≤ κ < 0.8: 적절한 일치 (수용 가능)
- κ < 0.6: 약한 일치 → 차원 정의 재논의 + Wayne 최종 판정

> 기록: `ml/results/manual_tag_irr.md` (분석은 데마 팀, 최지혜 보조)

### Wayne 검수 우선순위

1. 노란색 표시 cell (작업자 의심)
2. 이중태깅 불일치 cell
3. Lv4 도메인 위반 가능성 (M1·G3·M5)
4. 자가모순 흔적 (예: T1 TABLE인데 Q1 SHALLOW)

### LLM 1차 분류 옵션 (시간 부족 시)

P0 차원 일부는 ChatGPT/Claude로 1차 분류 후 사람 검수:
- 비용: 13 SKU × $0.01 = ~$0.20
- 가능 차원: T1·T2·T11·M3·M5·M6·G1·G2·G5·G7·Q3·Q5·Q6·Q8 (객관적 키워드 검색 위주)
- 사람 검수 5초/cell × 13 = 약 1분/차원

LLM 1차 사용 시 절차:
1. `python3 scripts/llm_first_pass_tagging.py --vertical medical_device,gargle` (스크립트 있을 시)
2. `data/processed/manual_tags_llm_draft.jsonl` 생성
3. 사람이 빠르게 검수 + Sheet에 입력
4. Step 4의 시간 50% 절감 가능

### 데마 (Phase B2/B3)
- features.jsonl + manual_tags.jsonl 좌측 join (sku_id 키)
- B2 baseline 로지스틱: 페이지 피처 + 수동 태깅 → 멘션 확률
- B3 XGBoost + SHAP: 차원별 기여도 시각화 + 워터폴
- 컨설팅 액션: SHAP 음의 기여 차원이 곧 *개선 우선순위*

### 산공통 (본실험 보류 해제 시)
- F1·F2·F4·F5·F6 가설 검정에 페이지 실측 차원 사용
- *바디닥터의 F 수준이 어디인지* 실측 매핑 → 본실험 컨디션과 비교
- transferability 검증 (가상 페이지 효과 vs 실제 페이지 효과)

### 컨설팅 (Phase C)
- 바디닥터 자사몰 진단 표 (F1~F6 + Sixthshop 점수 + Q1·Q9 외부 권위 등)
- Before/After 시뮬레이터 입력값
- 우선순위 액션 리스트의 정량 근거

## 10. FAQ

**Q: 5초 안에 결정 안 되면?**
A: 빈칸 + 노란색 + 슬랙 질문. 끌고 가지 말 것.

**Q: 자사몰 URL이 안 열리거나 로그인 벽이면?**
A: 슬랙 보고 — 박지윤·이소현이 fallback URL 결정 또는 그 SKU 그동안 빈칸.

**Q: M·G 도메인 차원인데 본인이 그 SKU 안 봐도 되나?**
A: 의료 SKU의 G 차원은 N/A 자동 (가글 SKU의 M도 마찬가지). 본인 vertical SKU만 입력.

**Q: 페르소나가 차원에 영향 주나?**
A: 수동 태깅은 *페이지 자체의 객관적 특성* — 페르소나 무관. 페르소나는 본실험 *쿼리*에 적용 (queries_*_최종.yaml).

**Q: 7명이 동시에 같은 시점에 봐야 하나?**
A: 네, 24~48시간 안에 모두 작업 권장. 페이지 변동(가격·재고·A/B 테스트) 영향 최소화. Day 2 한 번에 모이는 일정 추천.

**Q: 본 작업이 본실험 재실행과 어떻게 연결?**
A: 두 작업 *병렬*. 본실험 재실행은 LLM 호출(Y 변수 확보), 수동 태깅은 페이지 피처(X 변수 확보). 둘이 머지되어야 데마 모델 학습 가능.
