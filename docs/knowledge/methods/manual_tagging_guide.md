# 수동 태깅 가이드 — 페이지 정성 차원


> 각 차원마다 "**왜 이걸 보는가**" + "**무엇을 확인하나**" + "**5초 판정 팁**" + "**예시**" 4가지를 제공.


## 0. 태깅이 뭐고 왜 하는가 — 1분 설명

**태깅 = 페이지 안의 보이는 것/없는 것을 표 형식으로 기록하는 작업.**

우리는 상품 페이지를 모았고, 이걸 **"AI가 어떤 페이지를 추천 잘 해주는가?"** 분석하려고 함.
- AI 응답 데이터 (Phase B1) ➜ 어떤 브랜드가 추천받는지
- **태깅 데이터 (이 작업)** ➜ 그 브랜드 페이지가 어떤 특징인지
- 두 개를 합쳐서: **"페이지 A 특징 → 추천 확률 +X%" 같은 인사이트**

페이지 자동으로 뽑을 수 있는 건(JSON-LD 유무, 글자 수 등) 이미 추출 완료. **이 가이드는 사람이 봐야만 알 수 있는 정성 차원**만 다룸.

## 0.1 시작 전 체크리스트 (5분)

- [ ] `data/processed/manual_tags_template.csv` 열기 (Excel/Google Sheets)
- [ ] 본인이 맡은 태깅요소 범위 확인 (담당자별 분배)
- [ ] 본 가이드 한 번 훑기 (각 차원의 "이게 뭔지" 정도만)
- [ ] 첫 SKU 1개로 연습 (10분) → 막히는 곳 슬랙에 질문

## 0.2 태깅 절차 (한 SKU 약 1~2분)

1. 한 row 선택 → `raw_path` 컬럼의 HTML 파일 브라우저로 열기
   ```bash
   open data/raw/medical_device/bodydoctor/.../...rendered.html
   ```
2. **vertical 컬럼 확인**:
   - `medical_device` (의료기기 — 식약처 등록 요실금치료기): A + B + C 차원 답 (D는 N/A 자동)
   - `gargle` (가글): A + B + D 차원 답 (C는 N/A 자동)
   - `kegel_exerciser` (바디닥터 K · 닥터케이 등 — 의료기기 미등록 일반 운동기구): A + B + **C 일부**(M1·M2·M4·M6·M7 적용 / **M3 KFDA·M5 부작용은 N/A 자동**) — K는 의료기기 미등록이라 KFDA 등급 / 부작용 표기 의무 X
3. 페이지를 **위에서 아래로 한 번 빠르게 스크롤**하며 차원별로 판정
4. 5초 안에 결정 안 되면 **빈칸**으로 두고 넘어감 → 검수자(Wayne)가 일괄 결정
5. 한 SKU 끝나면 다음 row로

## 0.3 우선순위

# A. 페이지 구조·형식 (T1~T12)

> 페이지가 **어떻게 생겼는지**. 구조·형식 차원.

## T1. 주도 포맷 ★P0 (Format Dominance)

**왜 보는가**: AI는 표 형식 정보를 더 잘 파싱한다는 가설(F1) — 산공통 본실험과 직접 연결.

**무엇을 확인**: 페이지 본문이 **표(table) / 글머리(bullet) / 산문(paragraph) / 이미지** 중 무엇이 가장 많은 면적을 차지하는가.

**5초 판정 팁**: 페이지 위→아래 한 번 스크롤하면서 "이 페이지의 인상은 뭔가?" 직감.

**값**:
- `TABLE`: 스펙 표·비교 표가 본문 절반 이상
- `BULLET`: 글머리 기호(•)가 주도
- `PARAGRAPH`: 산문 단락 위주
- `IMAGE_HEAVY`: 텍스트보다 이미지 압도적 (긴 스크린샷 이어붙인 형태 — 한국 자사몰에서 흔함)

**예시**:
- 바디닥터 GN코스몰: TABLE (스펙 표 중심)
- 프로폴린스 자사몰: IMAGE_HEAVY (긴 비주얼 페이지)
- 코웨이 테라솔: PARAGRAPH (산문 + 이미지 혼합)

## T2. 인증 위치 ★P0 (Cert Position)

**왜 보는가**: 식약처/제허 번호의 시각적 위치가 신뢰감에 영향(F4 가설).

**무엇을 확인**: 식약처 인증·제허 번호·KFDA·CE 마크가 페이지의 어디쯤 보이는지.

**5초 판정 팁**: 페이지 처음 들어가서 첫 화면에 보이면 TOP, 한참 스크롤해야 보이면 BOTTOM.

**값**:
- `TOP`: 첫 화면 스크롤 안에 보임 (hero 영역, 상단 배지)
- `MIDDLE`: 중간 영역 (스펙 표 근처)
- `BOTTOM`: 페이지 하단·풋터·"제품 정보 고시"
- `NONE`: 페이지에 인증 정보 자체가 없음

**예시**:
- 이지케이: MIDDLE (스펙 표에 제허 번호 표기)
- 바디닥터 GN코스몰: BOTTOM (제품정보고시에만)
- 일부 공산품: NONE

## T3. 근거 유형 ★P0 (Evidence Type)

**왜 보는가**: AI가 추천 근거로 인용할 때 어떤 자료를 보고 판단하는가(F6 + H14).

**무엇을 확인**: 페이지가 "왜 이 제품을 사야 하는가"의 근거로 무엇을 가장 강하게 내세우는가.

**5초 판정 팁**: 페이지 본문에서 가장 자주 등장하는 "주장 + 그 옆 근거" 패턴 찾기.

**값**:
- `CLINICAL`: 임상 시험·논문·의사 인터뷰 인용 ("OO대학 임상에서…")
- `REVIEWS`: 사용자 후기·평점 강조 ("4.8/5 8,000건+")
- `MARKETING`: 카피 위주 ("특허받은", "최첨단", "혁신적인")
- `VISUAL_DEMO`: 시각 시연 (프로폴린스의 찌꺼기 시각화 같은 것)
- `NONE`: 근거가 거의 없음, 그냥 제품 소개만

## T4. 섹션 명확성 P1 (Section Clarity)

**왜 보는가**: 1차 발표 피드백 §2.2 — "각 세션이 명확하게 구분되면 추천 확률 ↑" 가설.

**무엇을 확인**: 스펙 / 사용법 / 리뷰 / FAQ 등 의미 단위가 시각적으로 명확히 구분돼 있는가.

**값**:
- `HIGH`: h2/section 헤더 명확 + 색·여백으로 구분 ("스펙" "리뷰" "FAQ" 명확 구분)
- `MID`: 일부만 구분, 일부는 섞임
- `LOW`: 한 덩어리로 흘러감 (긴 이미지 한 장으로 다 처리하는 경우)

## T5. 이미지 카테고리 P1 (Image Category)

**왜 보는가**: 1차 발표 피드백 §2.4 — 이미지 종류가 다른 효과(이미지 감성).

**무엇을 확인**: 메인 이미지(첫 화면 hero 이미지) 성격.

**값**:
- `LIFESTYLE`: 사용 장면·인물 (소파에 앉아 사용하는 모습)
- `PRODUCT_ONLY`: 제품 단독 컷 (흰 배경 등)
- `INFOGRAPHIC`: 수치·비교 그래픽 ("1주일 후 효과 ↑45%")
- `MIXED`: 두 가지 이상 섞임

## T6. 메시지 톤 P1 (Tone)

**왜 보는가**: AI는 신뢰감 있는 어조의 페이지를 선호할 수도(가설).

**값**:
- `EXPERT`: 의료 전문가 어휘·논조 ("저주파 자극을 통한 골반저근 강화")
- `FRIENDLY`: 일상어·구어체 ("산모님들에게 추천해요")
- `SALES`: 판매 강조 ("지금만 X% 할인" "한정 수량")

## T7. E-A-T 신호 강도 ★P0 (E-A-T Score 0~3)

**왜 보는가**: Google E-A-T (Expertise / Authoritativeness / Trustworthiness) 는 검색 랭킹의 핵심 + AI도 비슷한 신호를 봄(가설).

**무엇을 확인**: 3개 카테고리 각각에서 1점씩 체크.

**채점**:
- E (전문성): 의사·약사 인터뷰, 임상 인용, 학회 출처가 페이지에 있나? → +1
- A (권위): 식약처 번호, KFDA 인증, 정부 인증 마크, 대학병원 도입 → +1
- T (신뢰): 후기 100건+ 또는 평점 4.5+, 환불·배송 정책 명시 → +1

**합계**: 0 / 1 / 2 / 3

## T8. CTA 종류 P1 (Call to Action)

**왜 보는가**: 페이지의 비즈니스 모델이 추천에 영향. 렌탈 모델은 가격이 가려져 AI가 모를 가능성.

**값**:
- `BUY`: "구매하기" 버튼 위주
- `RENTAL`: "렌탈 신청" 강조 (코웨이 패턴)
- `INQUIRY`: "문의하기" 위주
- `CONSULT`: "상담받기" / "전문가 연결"

## T9. 비교 명시 P1 (Comparison Present)

**왜 보는가**: 자기 제품 vs 경쟁사 비교 표가 있으면 AI가 그 정보를 그대로 학습할 수 있음.

**값**: `YES` / `NO`

## T10. USP 명확성 ★P0 (USP Clarity)

**왜 보는가**: "왜 이 제품인가" 를 한 줄에 압축한 것 (USP) 이 첫 화면에 있는지 — 정보 hierarchy의 핵심.

**무엇을 확인**: 헤드라인 또는 서브헤드에 "이 제품을 선택해야 하는 한 줄 이유"가 명확히 보이는가.

**값**:
- `HIGH`: 헤드라인에 명확 (예: "출산 후 케겔, 1주 만에 변화" 또는 "프로폴린스 — 입 안의 찌꺼기가 보입니다")
- `MID`: 본문 중간에 있긴 함
- `LOW`: USP가 분산되어 있거나 없음, "그냥 좋은 제품" 수준

## T11. 가격 표기 P1 (Price Display)

**값**: `LIST` (정가만) / `DISCOUNT` (할인가 강조) / `RENTAL` (월 렌탈료) / `HIDDEN` ("문의" — 가격 비공개)

## T12. 개인화 신호 P1 (Personalization)

**무엇을 확인**: "산모용 / 전립선 수술 후 / 갱년기" 같이 페르소나별 분기 콘텐츠가 있는가.

**값**: `YES` / `NO`

## T13. 인증 마크 이미지 ★P0 (Cert Mark Visual)

**왜 보는가**: H5 (인증 명시 → 추천 ↑) 보강. 텍스트 키워드(자동 추출) 외에 **시각 인증 마크**가 페이지에 게시되어 있는지. AI는 이미지 alt-text 또는 인접 텍스트로 인지.

**무엇을 확인**: 식약처/KFDA/CE/GMP/ISO 등 인증 로고 이미지가 페이지에 표시되어 있는가.

**5초 판정 팁**: 페이지 상·하단·스펙 섹션 둘러보면서 정부/인증기관 로고 이미지 찾기.

**값**:
- `MULTIPLE`: 2개 이상 (예: 식약처 + KFDA + ISO)
- `SINGLE`: 1개
- `KEYWORD_ONLY`: 텍스트만 ("식약처 인증") 이미지 X
- `NONE`: 아무것도 없음

## T14. Answer-first 구조 ★P0 (Answer-first Structure)

**왜 보는가**: GN 측 피드백 (2026-04-26). AI는 컨텍스트 앞부분 우선 처리하는 경향이 있어, 페이지 첫 부분에 **핵심 답이 미리 요약**되어 있으면 AI 인용·추천 확률 ↑ 가설.

**무엇을 확인**: 페이지 첫 화면(스크롤 없이 보이는 영역)에 "이 제품은 X해서 Y한 분께 추천" 식의 답변/요약이 명시되어 있는가.

**5초 판정 팁**: 페이지 진입 직후 첫 화면 본문에 "결론·요약·핵심" 식 메시지가 보이는지.

**값**:
- `STRONG`: 첫 화면에 명확한 1~2 문장 결론·요약·USP 답변 형태
- `WEAK`: 일반 카피·이미지로 대체된 흐릿한 답
- `NONE`: 첫 화면이 hero 이미지 또는 카피만, 답변 형식 없음

## T15. Q&A 섹션 ★P0 (Q&A Section)

**왜 보는가**: GN 측 피드백 (2026-04-26). Q&A 섹션은 **AI가 학습된 패턴 그대로 인용 가능한 형식**. 자주 묻는 질문 + 답변 페어가 풍부하면 AI 답변에 반영 가능성 ↑.

**무엇을 확인**: 페이지에 "자주 묻는 질문" "FAQ" "Q&A" 형식의 명시적 섹션이 있는가.

**값**:
- `RICH`: 5개 이상의 Q&A 페어 명시적 섹션
- `BRIEF`: 1~4개 Q&A 페어
- `NONE`: 별도 Q&A 섹션 없음 (스펙 표·카피만)

# B. 공통 정성 (Q1~Q10)

> 페이지의 **콘텐츠 품질·신뢰성**. 의료기기·가글 둘 다 적용.

## Q1. 콘텐츠 깊이 ★P0 (Content Depth)

**왜 보는가**: AI는 깊이 있는 콘텐츠를 더 신뢰함 (Profound 측정 축 + Sixthshop B 카테고리).

**값**:
- `SHALLOW`: 광고 카피 위주, 한 줄 셀링 포인트 반복
- `MID`: 스펙·사용법 정도
- `DEEP`: 과학 원리·임상 데이터·상세 사용 가이드·FAQ 풍부

## Q2. 위기 상황 가이드 ★P0 (Safety Disclaimer)

**왜 보는가**: 책임 회피 + 신뢰 강화 차원 (의무 아님, 권장).

> ⚠️ **법적 의무는 M5 (부작용·금기)이고 Q2 자체는 권장입니다.** 이전 가이드 버전에서 Q2 NONE 시 "표시기준 위반 가능"으로 잘못 적었던 것을 정정합니다.

**무엇을 확인**: "X 증상이면 의료진 상담" 같은 안내가 있는가.

**값**:
- `EXPLICIT`: 명확 권고 ("증상이 지속되면 의사와 상담하세요")
- `IMPLICIT`: "지속되면 병원 방문" 정도 흐릿한 언급
- `NONE`: 안내 없음

## Q3. 학습 자료 P1 (Educational Resources)

**무엇을 확인**: 사용자가 별도로 볼 수 있는 PDF 매뉴얼·영상 가이드·운동 프로그램 등.

**값**: `RICH` (다중 자산) / `SOME` (1~2개) / `NONE`

## Q4. 소셜 프루프 종류 ★P0 (Social Proof Type) — 0~8 카운트

**왜 보는가**: 친구 대시보드 분석 — AI는 "X명이 보고 있음" 같은 사회적 증거에 영향.

**무엇을 확인**: 다음 8개 중 페이지에 있는 것 카운트.

체크리스트 (해당 모두 체크):
- [ ] 누적 판매량 ("X만 개 판매")
- [ ] 후기 수 (구체 숫자)
- [ ] 평점 (별점 4.5+ 등)
- [ ] 미디어 노출 ("OO신문" "유튜브 OO")
- [ ] 전문가 인터뷰 (의사/약사/연구원)
- [ ] 인플루언서·연예인
- [ ] 글로벌 판매 ("17개국 4,000만 병" 같은)
- [ ] 수상·인증 (KGC인증, 대한민국 명품 등)

**값**: 합계 0~8

## Q5. 환불·반품 정책 명확성 P1

**값**: `CLEAR` (정책 + 위생용품 제약 명시) / `PARTIAL` / `MISSING`

## Q6. 고객 지원 표기 P1

**값**: `RICH` (전화+채팅+이메일+운영시간) / `BASIC` (1~2 채널) / `NONE`

## Q7. 신뢰성 주장 톤 ★P0 (Trust Claim Tone)

**왜 보는가**: "왜 우리를 믿어야 하는가"의 어조가 AI의 인용 패턴에 영향.

**값**:
- `AUTHORITY`: 정부 인증·학회·전문가 인용 ("식약처 인증/대학병원 도입")
- `HERITAGE`: 역사·전통 ("130년 역사", "원조 일본")
- `INNOVATION`: 기술·특허 ("특허 출원", "AI 분석")
- `COMMUNITY`: 사용자 다수 ("X만 명이 선택")
- `MIXED`: 두 가지 이상
- `WEAK`: 명확한 신뢰 주장 없음

## Q8. 재구매 유도 메커니즘 P1

**값**: `YES` (정기배송/구독/멤버십) / `NO`

## Q9. 외부 권위 인용 ★P0 (External Authority Citation)

**왜 보는가**: H14 (외부 증거 → 추천 ↑) 보강. Q4 소셜 프루프(누적 판매·후기·인플루언서)와 별개로 **외부 권위(언론·논문·정부기관·전문 학회)** 인용을 별도 측정.

**무엇을 확인**: 페이지에 다음 같은 외부 권위 인용이 명시되어 있는가.

체크리스트 (해당 모두 체크):
- [ ] 신문 보도 (조선·중앙·매일경제 등)
- [ ] 방송 노출 (KBS·SBS·MBN·종편 등)
- [ ] 학술 논문 (저널명 + DOI/URL)
- [ ] 정부기관 (식약처·복지부·KOLAS 등 공식 인용)
- [ ] 의학회·전문 학회 (대한비뇨의학회 등)

**값**: 합계 0~5

## Q10. 수치 출처 유형 ★P0 (Numeric Source Type)

**왜 보는가**: H4 (수치 구체성 → 파싱 정확도) 보강. 자동 numeric_specificity_ratio는 "수치가 있는가" 만 봄. 이 차원은 **수치의 출처 신뢰도**까지 측정 — AI는 임상시험 출처 수치를 더 신뢰.

**무엇을 확인**: 페이지의 핵심 효능 수치(예: "85% 개선", "1주 후 효과")의 출처는?

**값**:
- `CLINICAL_TRIAL`: 임상시험 결과 (KCT 번호·논문 인용)
- `OWN_STAT`: 자체 통계 ("X만 명 사용 후 만족도")
- `CITED`: 외부 보고서·논문 인용 (3rd party)
- `MARKETING`: 출처 없는 카피 ("최대 효과", "혁신적인")
- `NO_SOURCE`: 수치 자체 없음

# C. 의료기기 특화 (M1~M7)

> **요실금치료기 페이지에만 적용** (vertical=medical_device).

## M1. 효능 주장 톤 ★P0 (Efficacy Claim)

**왜 보는가**: 식약처 의료기기 표시기준 + 의약외품·공산품 구분의 핵심.
- 의료기기는 "치료" "재활" 같은 의학적 효능 표현 정당하게 사용 가능
- 공산품(스탑요·페로니언 등)은 "운동기" "관리기" 표현만 허용 (의학적 효능 주장은 표시 위반)

**무엇을 확인**: 페이지가 효능을 어떤 톤으로 주장하는가.

**값**:
- `MEDICAL`: "치료" "재활" "회복" 의학적 효능 명시 (의료기기는 정당, 공산품이라면 위반)
- `FUNCTIONAL`: "운동" "강화" "관리" 기능적 표현만
- `LIFESTYLE`: "케어" "예방" "건강" 일반 표현
- `AMBIGUOUS`: 혼재

**해석 포인트**: 의료기기인데 LIFESTYLE이면 페이지가 자기 우위(의학적 권한)를 안 살림. 공산품인데 MEDICAL이면 잠재적 표시 위반.

## M2. 임상 근거 종류 ★P0 (Clinical Evidence Type)

**왜 보는가**: AI가 의학적 효능 주장을 뒷받침할 근거를 찾는다.

**값**:
- `OWN_TRIAL`: 자체 임상시험 결과 (KCT 번호·내부 데이터)
- `LITERATURE`: 학술지 인용 (논문명·저널 명시)
- `EXPERT`: 의사·약사 인터뷰 (영상 또는 글)
- `TESTIMONIAL`: 사용자 후기로 의학적 효능 주장 (회색지대 — 표시 위반 가능)
- `MARKETING`: "특허 기술" "최첨단" 카피만
- `NONE`: 근거 자료 없음

## M3. 식약처 인증 등급 명시 ★P0 (KFDA Class Display)

**왜 보는가**: 의료기기 1/2/3등급은 신뢰의 객관 척도.

**값**:
- `EXPLICIT`: "식약처 3등급 제허 X호" 명확히 표기
- `PARTIAL`: 등급 또는 번호 중 하나만
- `KEYWORD_ONLY`: "식약처 인증" 키워드만
- `NONE`: 표기 없음

## M4. 타겟 환자군 명시 ★P0 (Target Patient Specificity)

**왜 보는가**: 명확한 타겟이 있으면 AI가 페르소나 매칭에 활용.

**값**:
- `SPECIFIC`: "산후 6개월 이상 여성" "전립선 수술 후 회복기" 등 구체
- `GENERAL`: "골반저근 약한 분" "요실금 고민" 등 일반
- `MISSING`: 타겟 명시 없음

## M5. 부작용·주의사항 ★P0 (Adverse Effects Disclosure)

> ⚠️ **이 항목 MISSING은 의료기기 표시기준 위반 가능성**. 의료기기법 시행규칙 별표 — 사용설명서에 부작용·금기·주의사항 명시는 의무.

**값**:
- `FULL`: 별도 섹션 + 자세한 주의사항
- `BRIEF`: 1~2줄 짧은 경고
- `MISSING`: 부작용·금기 표기 없음

## M6. OEM/자체개발 표기 P1 (Manufacturing Origin)

**왜 보는가**: 화이트라벨 OEM 제품인지 자체 개발인지가 신뢰감에 영향. **바디닥터 OEM 의혹 핵심**.

**값**:
- `EXPLICIT_OWN`: 자체 R&D 명시, 제조사 동일
- `EXPLICIT_OEM`: OEM/ODM 표기 또는 다른 제조사 명시
- `AMBIGUOUS`: 제조사 모호 ("국내 제조" 정도)
- `MISSING`: 제조 정보 없음

## M7. 의료기관 도입 사례 P1 (Clinical Adoption)

**값**: `YES_NAMED` (기관명 명시 "서울대병원 OO과") / `YES_GENERIC` ("병원에서 사용중") / `NO`

# D. 가글 특화 (G1~G7)

> **가글 페이지에만 적용** (vertical=gargle).

## G1. 의약외품 분류 명시 ★P0 (Product Class Display)

**왜 보는가**: 가글은 식약처 분류상 **의약외품 / 일반 화장품 / 식품** 중 하나. 분류에 따라 허용 효능 표현이 다름.

**값**:
- `EXPLICIT`: "의약외품" 명확 표기 (또는 다른 분류)
- `KEYWORD`: "구강위생용품" 정도 흐릿
- `MISSING`: 분류 표기 없음

## G2. 유효 성분 함량 ★P0 (Active Ingredient Concentration)

**왜 보는가**: AI는 구체 수치를 주장의 신뢰도로 활용 (가설 H4).

**예시 성분**: CPC(세틸피리디늄클로라이드), 클로르헥시딘(CHG), 불소(ppm), 알코올(%)

**값**:
- `FULL`: 모든 유효 성분 + 함량 명시
- `PARTIAL`: 성분명만, 함량 미표기
- `KEYWORD`: "특수 성분" 마케팅 표현만
- `NONE`: 성분 정보 없음

## G3. 효능 주장 범위 ★P0 (Efficacy Scope)

**왜 보는가**: 의약외품 표시광고 가이드라인 — 가글은 허가받은 효능만 광고 가능.
- 허가된 효능: 구취 제거, 치아 청결
- 의약외품 추가 효능: 잇몸 질환 예방, 충치 예방
- **금지 효능**: "잇몸 치료", "충치 치료" (의약품 영역 침범)

**값**:
- `LICENSED_ONLY`: 허가 효능만
- `EXPANDED`: 의약외품 추가 효능까지
- `OVERREACH`: 의약품 영역 침범 (위반 가능)
- `AMBIGUOUS`: 모호

## G4. 시각적 효능 시연 ★P0 (Visual Efficacy Demo)

**왜 보는가**: 프로폴린스 USP 핵심 — "찌꺼기 시각화"는 AI가 텍스트로 학습 가능한 시그널.

**값**:
- `STRONG`: 색 변화·찌꺼기·미생물 등 시각적 직관 시연
- `WEAK`: 일반 사용 장면
- `NONE`: 시각 시연 없음

## G5. 알코올·민감 성분 표기 P1

**값**: `EXPLICIT` ("무알코올" / "X% 알코올") / `MENTIONED` (성분표만) / `MISSING`

## G6. 사용 빈도·방법 권고 P1 (Usage Recommendation)

**값**: `DETAILED` (빈도·시간·후속 행동 모두 안내) / `BRIEF` ("양치 후") / `MISSING`

## G7. 원산지·수입품 표기 P1 (Origin Display)

**값**: `EXPLICIT` (원산지 + 제조국 명확) / `BRAND_ONLY` (브랜드 국적만) / `MISSING`

# F. 검수·수렴

- 팀 3명이 같은 SKU 5~10개를 **이중 태깅** → inter-rater agreement (Cohen's kappa ≥ 0.6 목표)
- 불일치 항목은 Wayne 최종 판정
- 결과 저장: `data/processed/manual_tags.jsonl`

# G. 모델·컨설팅 통합

- features.jsonl + manual_tags.jsonl 좌측 join (sku_id key)
- 데마 B2/B3 모델 입력 (categorical → one-hot)
- SHAP에서 "수동 태깅 차원 vs 자동 차원" 기여도 비교
- 컨설팅 단계: 정성 차원이 **개선 액션 직접 연결** (예: "M2 임상 근거 NONE → 임상 시험 발주 권고")

# H. LLM 1차 분류 옵션 (반자동)

P1 차원 일부는 ChatGPT/Claude에 페이지 텍스트 + 차원 질문 던져 1차 분류 후 사람 검수가 효율적.
- 비용: 47 SKU × $0.01 = ~$0.50
- 사람 검수 5초/판정 × 47 = 4분/차원
- P1 차원 17개 적용 시 약 1시간

가능 차원 (객관적 판정): T1, T2, T11, Q3, Q5, Q6, Q8, M3, M5, M6, M7, G1, G2, G5, G6, G7
어려운 차원 (페이지 시각 인상 필요): T5, T6, T10, Q4, Q7, M1, M4, G3, G4 — 사람 권장
