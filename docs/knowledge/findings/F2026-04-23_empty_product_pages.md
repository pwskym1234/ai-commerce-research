## 의미

**식약처 3등급 의료기기인데 양사 모두 페이지에 그 사실을 전혀 알리지 않음.**

이는 단순한 "F4=하단" 또는 "F5=인증완료만"보다 더 극단적인 상태:
- F1 HTML 포맷: 평가 불가 (스펙 자체 없음)
- F2 JSON-LD: none
- F3 수치 구체성: none (수치 자체 없음)
- F4 인증 위치: **none**
- F5 인증 상세: **none**
- F6 임상 근거: **none**

→ 우리 가상 페이지 F4·F5·F6의 "없음" 수준에 정확히 대응하는 실제 사례.

## 가설 검정에 미치는 영향

### H1~H9 모두에 강한 사전 신호
- **H1**: HTML 포맷 효과 — 양사 페이지가 이미지 중심이라 LLM은 텍스트 거의 없는 상태에서 평가
- **H4**: 수치 구체성 — 둘 다 "none" → 가상 페이지에서 "정확/모호" 변형 효과가 더 두드러질 것
- **H5**: 인증 명시 — 둘 다 "없음" 상태가 baseline → "등급+허가번호 풀 명시" 도입 효과 측정 시 효과 크기 ↑
- **H7**: 임상 인용 — 양사 모두 "없음" → "임상 인용" 변형의 marginal benefit ↑
- **H8 (Y1≠Y2 검정의 강한 사례)**: 페이지에 정보 거의 없으므로 Y1(파싱 정확도) 매우 낮을 것 → 그럼에도 AI가 Y2(추천)을 어떻게 결정하는가? **외부 학습 데이터 의존? Y4(안전성 회피)로 도망?**
- **H10**: 페이지 정보 부족하면 AI가 의료기기/비의료기기 구분 더 못할 것

## ⚠️ 정정 (2026-04-23 크롤러 검증 후)

WebFetch로 본 결과("텍스트 거의 없음")는 **LLM이 페이지 일부만 보고 요약한 한계**였음. 실제 크롤러로 추출한 피처:

| 항목 | WebFetch 추정 | 크롤러 실측 (page 187) |
|------|------------|---------------------|
| text_length | "거의 없음" | **2,591 글자** |
| 이미지 | "이미지 중심" | 19개 |
| TABLE | "없음" | **5개** |
| BULLET (li) | "없음" | **54개** |
| JSON-LD | "없음" | **Product 스키마 있음** (필드 7개) |
| 인증 키워드 | "전혀 없음" | **5번 등장** (위치: 122번째 글자) |
| 등급 명시 (3등급/2등급) | — | **없음** ✓ (최초 추정 일치) |
| 임상 키워드 | "없음" | **0** ✓ |
| 정확 수치 | "없음" | **0** ✓ (한국어 정규식 매칭 약할 수 있음 — 수동 점검 필요) |

**수정된 결론**:
- ✅ **여전히 진실인 부분**: 식약처 3등급 명시 없음, 임상 데이터 인용 없음, FDA 등급 명시 없음
- ⚠️ **수정**: 페이지가 "텅 빈" 게 아니라 **"이미지 중심이지만 일정 텍스트 + Product JSON-LD + 인증 키워드 일부는 있음"**. 다만 의료기기 특화 정보(등급/허가번호/임상)가 약함

이 정정은 컨설팅 임팩트의 크기를 약간 줄이지만, **여전히 F4·F5·F6 강화는 큰 효과 예상**.

**일반 교훈**: WebFetch는 정보 추출용 빠른 도구지만, **정확한 피처 측정은 크롤러 + BeautifulSoup 파싱**이 필요. 향후 진단은 둘 다 사용.
