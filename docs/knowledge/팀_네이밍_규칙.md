# 팀 네이밍 규칙 — 신규 팀원도 5분 안에 이해되도록

**작성**: 2026-04-30 (팀 피드백 반영, 2026-04-30 Rule 6 추가)
**적용 범위**: 새 파일·새 코드·새 문서·기존 파일 정리 시.
**목적**: "v5가 뭐예요?", "BRD가 뭐예요?", "F2가 뭐예요?" 라는 *반복되는 질문*을 없애고, 팀원이 *어떤 파일이 최신인지* 1초 안에 알 수 있게.

---

## 1. 왜 어려웠는가 (원인 진단)

팀원이 헷갈리는 패턴 3가지:

### 원인 1 — 버전 번호가 *의미 없음*
**과거 사례**: `queries_medical_v5.yaml` (옛 이름)
- v5가 v4보다 뭐가 다른지 파일명만 봐선 모름
- 파일을 열어서 주석을 읽어야 비로소 차이를 앎
- *원래 작성자(Wayne·Claude)는 알지만 새 팀원은 모름*

### 원인 2 — 영어 약어가 *글로서리 부재*
**예**: BRD / CAT / SYM / CMP / COM / PRC / USE / DEC (쿼리 8유형)
- 산공통/데마 팀원이 이 코드를 처음 봤을 때 의미를 모름
- 코드는 *원작자의 머릿속*에서 자연스럽지만 *팀에게는 외계어*

### 원인 3 — 코드 letter shortcut이 *맥락 없이 등장*
**예**: F1, F2, ..., F6 / H1, H2, ..., H15 / Y1, Y2, ..., Y4 / R1, R2, ..., R10 / L18, L36 / Phase A/B/C/D / B1, B2, B3
- 이 모든 코드가 *서로 다른* 분류 체계 (요인 / 가설 / 종속변수 / 룰 / 직교배열 / 페이즈)
- 같은 letter (B, F)가 다른 의미로 재활용되어 더 헷갈림

### 핵심 통찰

**코드는 *원작자의 두뇌*에 효율적이지만 *팀의 두뇌*에는 비효율적**. 코드 자체를 없앨 필요는 없음 — 다만 *처음 보는 사람이 즉시 이해할 수 있는* 한국어 bridge가 필수.

## 2. 새 파일 작성 룰 6개

### Rule 1 — 파일명은 *의미 기반*, 버전 번호 X

❌ **나쁨**: `queries_medical_v5.yaml`
- "v5가 뭐?" 라는 질문 즉시 발생

✅ **좋음**: `queries_medical_네이버검색반영.yaml` 또는 `queries_medical_2026-04-30.yaml`
- *변경 사유*가 파일명에 보임 → 무엇이 다른지 즉시 이해

### Rule 2 — 버전 번호 대신 *의미 접미사*

새 파일이 기존 파일을 *대체*가 아니라 *별도 트랙*인 경우:
- `_원본` (preservation)
- `_초안` (draft)
- `_검수전` / `_검수후`
- `_검색반영` (specific change)
- `_경쟁사반영`
- `_B1보존` (특정 마일스톤 정합 보존)
- `_최종` (현재 가장 최신 — Rule 6)

날짜 접미사도 가능: `_2026-04-30`

❌ **나쁨**: `queries_medical_v5.yaml`
✅ **좋음**: `queries_medical_네이버검색반영.yaml`
✅ **좋음**: `bodydoctor_pdp_copy_초안.md` (옛 `_draft` 영어 → 한국어)

### Rule 3 — 영어 약어는 *데이터에만* 유지, 문서엔 한국어 우선

데이터 파일 (jsonl, csv, yaml의 식별자):
- BRD, F2, H10, R1 등 *그대로 유지* (분석 코드 호환성)
- 영어 약어는 *간결한 라벨*로서의 가치 있음

마크다운 문서:
- 처음 등장 시 *한국어 풀이름 + (영어 약어)* 로 표기
- 한 번 풀이름 적었으면 이후 약어 사용 OK

❌ **나쁨**: "BRD 쿼리에서 anchor mention 21%"
✅ **좋음**: "브랜드 지명(BRD) 쿼리에서 anchor mention 21%"

### Rule 4 — Glossary 의무 인덱싱

새 코드를 만들 때 *반드시*:
1. `docs/knowledge/glossary.md` 에 한국어 한 줄 정의 추가
2. 처음 사용한 문서에 *Glossary 링크* 또는 *인라인 풀이름*

코드를 글로서리에 없이 새로 도입하면 *팀의 헷갈림 부채*가 누적됨.

### Rule 5 — 한국어 파일명 OK

- macOS / Linux / Git 모두 UTF-8 한국어 파일명 지원
- 기존 `findings/AiEO_경쟁사_리서치.md` 같이 이미 사용 중
- 단 *코드 파일* (.py, .yaml 식별자, .json 키)은 ASCII 유지 — 도구 호환성

### Rule 6 — 가장 최신 = `_최종` 라벨, 이전 버전 = `_최종` 표시 제거 (2026-04-30 신설)

**Wayne 룰 (2026-04-30)**: "v1/v2/v4/v5 같은 버전 번호 X. 가장 최신이면 *이전 버전에 있던 최종 이름을 지우고* 가장 최근 버전에 *최종*이라는 이름을 붙여. 모든 것에 적용."

**적용 사례 (2026-04-30 일괄 정리)**:

| 옛 이름 (v 시리즈) | 새 이름 (의미 + 최종 표기) |
|---|---|
| `queries_medical.yaml` (v4) | `queries_medical_B1보존.yaml` — Phase B1 결과 정합성 보존용 |
| `queries_medical_v5.yaml` | `queries_medical_네이버검색반영.yaml` — NAVER 검색 데이터 보강 단계 |
| `queries_medical_페르소나반영.yaml` | `queries_medical_최종.yaml` — 가장 최신, 페르소나 stratified 32 쿼리 |
| `queries_gargle.yaml` (v2) | `queries_gargle_B1보존.yaml` |
| `queries_gargle_v5.yaml` | `queries_gargle_네이버검색반영.yaml` |
| `queries_gargle_페르소나반영.yaml` | `queries_gargle_최종.yaml` |
| `queries_kdr.yaml` (v2) | `queries_kdr_최종.yaml` |
| `experiment_design_v2.md` | `experiment_design.md` (단일 파일 = 최신 = v 표기 불필요) |
| `bodydoctor_pdp_copy_draft.md` (v0.1) | `bodydoctor_pdp_copy_초안.md` |
| `propolinse_pdp_copy_draft.md` (v0.1) | `propolinse_pdp_copy_초안.md` |

**원칙**:
1. **현재 가장 최신** = `_최종` 라벨 부여
2. **이전 버전들** = `_최종` 표시 제거 + *목적/단계 의미 라벨* 부여 (`_B1보존`·`_네이버검색반영`·`_초안` 등)
3. **DIGEST.md 같은 단일 파일**(매주 갱신) = v 번호 자체 무의미 → "마지막 갱신: 2026-04-30" 만, "v2.8" 표기 X
4. **본문 안의 v 표기 (v0.1·v1·v3→v4 같은 변경 이력 인용)** = 의미 명확하면 (예: 변경점 요약) 유지 OK, 단순 "v 몇" 표기는 제거
5. **신규 작성** 시: 단일 파일이면 `_최종` 안 붙임 (기본 = 최신). 별도 트랙이 추가되면 그때 `_최종` 라벨 붙임.

**왜**:
- 팀원이 *어떤 파일이 현재 사용되는지* 파일명만 봐도 1초 안에 앎
- v 번호의 *의미* 추적 부담 없음 (변경 사유는 파일 안 주석으로)
- *최종*이 항상 *현재 사용 가능한 가장 좋은 것*이라는 단순 약속

## 3. 기존 파일은 어떻게

**원칙적으로 rename 가능** — Rule 6 적용 (2026-04-30 일괄 정리). 단:
- rename 시 *모든 참조 갱신* (sed batch + 검증 grep)
- git mv 사용 (history 보존)
- 한 commit에 mv + 참조 갱신 함께 (intermediate 깨진 상태 회피)

대규모 rename이 어려우면:
- 매 파일 *상단 주석/frontmatter*에 "현재 최신 = X" 명시
- `glossary.md` 의 "Versions" 섹션에 모든 옛 이름 의미 표

## 4. 자주 헷갈리는 코드 → 한국어 cheat sheet

### 쿼리 유형 (8유형)

| 코드 | 한국어 풀이름 | 예시 |
|---|---|---|
| BRD | 브랜드 지명 (Brand-Named) | "바디닥터 효과 있나?" |

| SYM | 증상 기반 (Symptom-Based) | "산후 요실금에 뭐 좋아?" |
| CMP | 비교 (Comparative) | "바디닥터 vs 이지케이" |

| DEC | 단일 선택 강제 (Decision) | "딱 하나만 골라줘" |

### 산공통 통제 요인 (F1~F6)

| 코드 | 한국어 풀이름 | 의미 |

## 6. 컨벤션 적용 우선순위

신규 작성 시 100% 적용:
- 새 파일 (오늘부터)

- 새 코드 docstring·주석

- 문서 수정 시 첫 등장 코드에 한국어 풀이름 추가

## 7. 새 코드를 도입할 때

새 letter shortcut(F7, H16, Y5, R11 등)을 도입할 때:

1. **반드시 glossary.md 에 추가** (한 줄 정의 + 풀이름)
