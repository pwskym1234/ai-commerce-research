# 친구 대시보드 리뷰 — 양방향 정리

## A. 우리 프로젝트 기준 → 친구 대시보드가 바꿔야 할 것 (권고)

우리 기준 = [methods/experiment_design.md](../methods/experiment_design.md) + [CLAUDE.md §4 R1~R10 재현성 룰](../../../CLAUDE.md)

### A.1 반복 횟수 2회 → **20회 이상 (R1 최소 기준)** 🔴 P0

**현재**: 149 프롬프트 × 2회 = 298건
**문제**: n=2에서 Wilson 95% CI가 터무니없이 넓음
- 프롬프트 개별 "추천률 0%" 같은 수치는 **n=2에서 신뢰구간 [0%, 84%]**
- "긍정 비율 6.7%" 같은 브랜드 지명 30건 기준도 **95% CI [1.1%, 21.4%]**

**권고**:
- 최소 **n=20** (우리 R1 기준, Wilson 95% CI ±0.21)
- 이상적 **n=30~50** (±0.14~0.18)
- 149 프롬프트 × 20회 = 2,980호출. gpt-5.4-nano 기준 **약 $3** — 비용 부담 없음
- 구현: runner 루프에서 `n_repeat = 20` 강제

### A.2 모델 버전 / temperature / seed 메타 미공개 🔴 P0 (R2/R3/R5)

**현재**: 대시보드 상단에 "감사 실행일" 만 있고 모델/하이퍼파라미터 없음
**문제**: 재현성 불가. GPT-4o 인지 gpt-5.4인지 Claude인지 모름
**권고**: 상단 헤더에 다음 강제 표기:
```
모델: gpt-5.4-2026-04-15 · temperature: 0.7 · seed: 42~62 (rep별)
```
또는 각 응답의 jsonl 메타에 `model_version`, `temperature`, `seed` 필수 기록 후 대시보드에서 요약.

### A.3 Position bias 셔플 증거 없음 🟡 P1 (R4)

**현재**: 경쟁사 리스트 순서가 매 호출마다 랜덤 셔플됐는지 불명
**권고**: 
- 각 jsonl 라인에 `products_order: [brand1, brand2, ...]` 기록 (우리 runner.py 이미 구현)
- 대시보드 메타 섹션에 "순서 셔플: seed 기반 매 반복 랜덤" 명시

### A.4 신뢰구간 (Wilson 95% CI) 누락 🔴 P0 (R7)

**현재**: 모든 지표가 점추정치(%)만. 오차 막대 없음
**문제**: "추천률 0.0%"와 "추천률 48%"가 **나란히 똑같이 정확한 숫자처럼 보임**. 전자는 n=148에서 CI [0%, 2.5%], 후자는 n=60에서 CI [35.7%, 60.8%] — 완전히 다른 신뢰도
**권고**:
- 모든 비율 지표 옆에 회색 error bar 표시
- Wilson score CI 사용 (이항 분포, 작은 n에서도 안정)
- 호버 시 `95% CI [a, b]` 툴팁

### A.5 쿼리 유형 4개 → 우리 8유형 매핑 필요 🟡 P1

**현재**: 중립(148) / 브랜드지명(30) / 경쟁사대안(60) / 직접비교(60)
**우리 8유형**: BRD / CAT / SYM / CMP / COM / PRC / USE / DEC

친구 4유형 ⊆ 우리 8유형 대응:
- 친구 "중립" = 우리 CAT + SYM + PRC + USE 혼합 (4개가 합쳐짐)
- 친구 "브랜드지명" = 우리 BRD
- 친구 "경쟁사대안" = 우리 COM
- 친구 "직접비교" = 우리 CMP + DEC

**권고**: 친구 "중립" 148건을 **CAT/SYM/PRC/USE 4개로 서브분류**. 우리 H11 (쿼리에 "의료기기" 명시 효과)는 친구 중립 148건 안에서 **키워드 유무 필터**로 관찰 가능. 샘플 수 확보됨.

### A.6 감성 분류기 스펙 명시 🟡 P1

**현재**: 브랜드 지명 30건 중 중립 93.3% — 분류기가 너무 보수적으로 "중립" 기본값에 쏠렸을 가능성
**권고**:
- 감성 분류에 쓴 모델/프롬프트/few-shot 예시 공개
- 검증 샘플 (예: 100건 수동 라벨) 대비 분류기 정확도 보고
- 분류 불확실 응답은 "모호" 카테고리 추가 (강제 3분류 대신)

### A.7 Raw 응답 접근 / 다운로드 미지원 🟡 P1 (R8)

**현재**: "상세 보기 →" 링크는 있으나 jsonl 공개 여부 불명
**권고**:
- 각 감사 run의 raw jsonl 다운로드 버튼
- 필드: run_id, timestamp, model, seed, system_prompt_hash, user_prompt, raw_response, parsed_y 등
- 재현성 + 외부 검증 가능

### A.8 응답 캐시 상태 표시 🟢 P2 (R10 비슷한 원칙)

대시보드가 같은 프롬프트를 여러 번 호출하면 **OpenAI 과금 낭비**. 캐시 메커니즘 표시 권고:
- "이번 run의 캐시 히트 12%" 같이 대시보드 요약
- 우리 runner.py에 이미 구현되어 있음

### A.9 PRC / USE / DEC 쿼리 유형 신규 추가 🟡 P1

현재 대시보드엔 **가격형·유스케이스형·구매의도형 쿼리가 없음**. 우리 8유형 중 3개 누락.
**권고**: 각 6개씩 총 18 프롬프트 신규 추가:
- PRC: "1만원대 가글 추천", "프리미엄 10만원대 가글"
- USE: "출산 후 회복용 가글", "술자리 전 쓸 가글"
- DEC: "하나만 골라줘" (단일 선택 강제)

### A.10 F 요인 매핑을 위한 메타 필드 (장기) 🟢 P2

대시보드 문제→원인→액션 3쌍이 자동 생성인데, **F 요인과 연결 안 됨**. 본실험 결과가 나오면 "F5=Full 적용 시 중립 언급률 +X%" 같은 정량 예측 추가 가능.
**권고**: 각 프롬프트에 `faction_map` 메타 (어느 F가 이 쿼리에서 중요한지 사전 라벨링). 장기 개선.

### A.11 재측정 주기 / 이전 run과의 비교 🟢 P2

"히스토리 1건"만 있고 추이 없음. 월간 재측정 체계 필요.
**권고**: 
- 매월 자동 재실행 (cron)
- 이전 run과 diff (추천률 ±, 신규 발굴 경쟁사 등)
- 개선 액션 실행 전/후 측정 A/B

### A.12 다중 모델 비교 탭 🟢 P2

**현재**: 1개 모델만 가정
**우리 계획**: gpt-5.4-nano 본실험 + 발표 전 Gemini/Perplexity sanity check
**권고**: 대시보드에 "모델별 탭" 추가. 같은 쿼리를 여러 모델에 돌려 답변 일관성 비교.

### 요약 — 우선순위

| 우선순위 | 항목 |
|--------|------|
| 🔴 **P0** | 반복 20회 + 모델/temp/seed 메타 공개 + Wilson CI |
| 🟡 **P1** | 4유형 → 서브분류 + 감성 분류기 스펙 + raw 접근 + PRC/USE/DEC 신규 |
| 🟢 **P2** | 캐시 상태 + F 매핑 + 재측정 추이 + 다중 모델 |

## B. 친구 대시보드에서 우리 프로젝트에 가져올 좋은 것

### B.1 "기타" 카테고리 자동 발굴 ★★★

친구 대시보드가 AI 응답 텍스트에서 **브랜드 NER로 미인지 경쟁사를 자동 추출**. 프로폴린스 경우:
- 우리 기존 리스트: 리스테린/가그린/2080/페리오
- 친구가 발견: **오랄비·센소다인·콜게이트 플락스·테라브레스·어썸쿨 프로폴리스 가글 600ml·광동 인후엔·리브러쉬 클러브·유시몰**
- 특히 **어썸쿨 프로폴리스** — "프로폴리스" 계열 직접 경쟁자. 우리가 놓침.

**우리 프로젝트 반영**:
- runner.py의 [`parse_y2a`](../../../experiments/runner.py) 에 **사전 정의 경쟁사 외에 응답 텍스트 NER** 추가
- jsonl 필드 신규: `other_brands_detected: ["오랄비", "센소다인", ...]`
- 본실험 후 분석: "우리가 정의한 N=6 외에 AI가 실제로 추천하는 브랜드" 리포트
- 구현: 간단한 정규식 + 한국어 브랜드 사전 + LLM 분류기

### B.2 쿼리 유형별 "맞춤 지표" 설계 ★★★

친구 대시보드의 강점: **각 유형마다 고유의 측정 지표**:
- 중립 → **언급률 + 추천률** (AI prior 진입)
- 브랜드지명 → **긍정/중립/부정 톤** (감성)
- 경쟁사대안 → **대안 소환율 + 미노출률** (대체재 흡수력)
- 직접비교 → **승/무/패** (경쟁 포지셔닝)

**우리 프로젝트 반영**: 
- 우리 Y2a는 "멘션 여부" 단일. 쿼리 유형별로 의미가 달라짐
- 신규 파생 지표 추가 (runner 응답 파싱 보강):
  - **Y2a_mention** (중립 쿼리): 멘션 여부
  - **Y2a_positive** (BRD 쿼리): 긍정 감성 여부
  - **Y2a_alternative** (COM 쿼리): 대안으로 소환됐는지 + 미노출 여부
  - **Y2a_wintieloss** (CMP 쿼리): {win, tie, loss} 3분
  - **Y2a_singleselect** (DEC 쿼리): 최종 선택 받았는지 binary
- → 기존 Y2a "포괄적" 정의에서 **쿼리 유형별 특화 Y** 로 진화
- 분석에서 H5/H7 등 각 가설별로 **적합한 Y 변수** 사용 → 검정력·해석력 ↑

### B.3 "문제 / 원인 가설 / 바로 할 일" 3쌍 구조 ★★

친구 대시보드의 "우선 개선 과제" 섹션:
```
1. 문제: 리스테린 대안 질문에서 우리 브랜드 미노출 100%
   원인 가설: 경쟁사 대체재 맥락에서 AI가 우리 브랜드를 후보로 인식 못 함
   바로 할 일: ✓ "리스테린 대신 우리 브랜드" 비교 콘텐츠 작성 ...
```

**우리 프로젝트 반영**:
- action_roadmap.md의 "Before/After" 섹션을 이 구조로 **재구성**
- 발표 슬라이드: "결과"가 아니라 "문제→원인→액션" 3쌍으로
- H 검정 결과가 나오면 각 문제의 원인이 통계적으로 입증됨

### B.4 약점 프롬프트 / 강점 프롬프트 명시적 리스트 ★★

친구 대시보드가 "전혀 언급되지 않은 질문 top 3" / "자주 언급된 질문 top 3" 직접 표시.
**우리 프로젝트 반영**:
- h_tests.py에 추가: 본실험 후 **하위 10% Y2a 쿼리**의 원문 + F 조합 리포트
- 발표 슬라이드: "이 쿼리에서 0% 선택" 같은 구체 사례로 청중 몰입 높이기

### B.5 "기타" 경쟁사 자동 확장 → 우리 데마 피처 강화 ★★

친구 대시보드가 **미인지 경쟁사 자동 발굴**하면서 그 브랜드들의 점유율까지 계산. 우리 데마 분석에 직접 영향:
- 현재 우리 데마는 **사전 정의 N=6** 브랜드 피처만 사용
- 친구 방식: 응답에서 등장 빈도 집계 → **10위 테이블** 생성
- 우리 반영: 데마의 external_evidence 변수에 "응답 등장 타 브랜드 수" 추가. H14 (외부 증거) 관찰 변수로 사용.

### B.6 쿼리 문구 스타일 — 자연 유저 톤 ★

친구 쿼리: "구강청결제 처음 써보는데 뭐 사면 돼?" / "가글 입문자한테 무난한 거 뭐 있어?"
우리 쿼리 (queries.yaml): "가정에서 쓸 수 있는 케겔운동기 추천해주세요." (존댓말 + 완성문)

**차이**: 친구는 **짧고 구어체**, 우리는 **정돈된 서술문**. 실제 사용자 쿼리 분포는 전자에 가까움.

**우리 프로젝트 반영**:
- queries.yaml 24개 중 **절반을 구어체·짧은 문장**으로 바꿈
- 유형당 3개 = (존댓말 긴 문장 1 + 반말 짧은 문장 1 + 불완전 문장 1) 구성
- 쿼리 문구 다양성 확보 → H15 (Q×F 교호작용) 안정성 ↑

### B.7 "경쟁사 × 질문 수" 그리드 ★

친구 대시보드의 경쟁사 대안·직접 비교 테이블:
```
경쟁사           질문 수   대안 소환율   공동 언급률
리스테린         20        0.0%         0.0%
어썸쿨 프로폴리스 20        25.0%        5.0%
테라브레스       20        0.0%         0.0%
```

이런 **"경쟁사별 세로, 지표 가로" 그리드가 분석 직관성 높음**.
**우리 프로젝트 반영**:
- ml/scripts/eda.py에 `competitor_x_metric_grid` 함수 추가
- 각 경쟁사별 우리 Y2a/Y5 지표 한 번에 조망

### B.8 "N건의 AI 응답 분석" 헤더 포맷 ★

대시보드 상단이 항상 **"총 N건의 AI 응답을 분석했습니다"** 로 시작 — 신뢰성·규모감 주는 표현.
**우리 반영**: 발표 첫 슬라이드도 같은 톤으로. "본실험은 25,920건의 gpt-5.4 응답을 54개 가상 페이지 × 24 쿼리 × 20 반복으로 생성했습니다."

## A. 친구 대시보드가 우리 기준으로 바꿔야 할 것

우리 기준 = [experiment_design.md](docs/knowledge/methods/experiment_design.md) + [CLAUDE.md §4 R1~R10 재현성 룰](CLAUDE.md)

### 🔴 P0 (즉시)

**A.1 반복 2회 → 최소 20회 (R1)**
- 현재 149 프롬프트 × 2회 = 298건 → n=2에서 "추천률 0%"의 Wilson 95% CI가 **[0%, 84%]**. 사실상 "모름"
- "긍정 6.7%"도 n=30에서 CI **[1.1%, 21.4%]** — 점수 자체가 신뢰 불가
- 권고: n=20 최소. 149 × 20 = 2,980호출, nano 기준 **$3**. 비용 부담 전무.

**A.2 모델 버전·temperature·seed 메타 누락 (R2/R3/R5)**
- 대시보드 헤더에 모델/하이퍼파라미터 아무것도 없음 — gpt-4o인지 gpt-5.4인지 불명
- 권고: 헤더에 강제 표기
```
모델: gpt-5.4-2026-04-15 · temperature: 0.7 · seed: 42~62 (rep별)
```

**A.3 Wilson 95% CI error bar 전혀 없음 (R7)**
- "추천률 0.0%"와 "추천률 48%"가 똑같이 정확한 숫자처럼 보임. 전자는 n=148, 후자는 n=60으로 CI 폭 완전히 다름
- 권고: 모든 비율 지표에 회색 error bar + 호버 시 `95% CI [a, b]` 툴팁

### 🟡 P1

**A.4 Position bias 셔플 증거 없음 (R4)**
- 경쟁사 리스트 순서를 매 호출마다 셔플했는지 불명
- 권고: jsonl에 `products_order` 기록 (우리 runner.py 이미 구현) + 대시보드 메타 섹션에 명시

**A.5 친구 4유형 → 우리 8유형 매핑**
- 친구 "중립" 148건 = 우리 CAT+SYM+PRC+USE 혼합 (4개가 합쳐짐)
- 권고: "중립" 148건을 4개 서브카테고리로 라벨링. H11 (의료기기 키워드 명시 효과) 관찰 가능.

**A.6 감성 분류기 스펙 불명**
- 30건 중 중립 93.3% — 분류기가 "중립" 기본값에 쏠렸을 가능성
- 권고: 분류 모델/프롬프트/few-shot 공개 + 100건 수동 라벨 대비 정확도 보고

**A.7 Raw jsonl 다운로드 미지원 (R8)**
- 외부 검증·재현성 위해 각 run의 raw jsonl 공개 필수

- 권고: 각 6개씩 18개 신규 추가

## B. 친구 대시보드 → 우리가 반영할 좋은 지점

### ★★★ B.1 "기타" 카테고리 자동 발굴 — **우리가 놓친 경쟁사 다수**

친구 대시보드가 응답 텍스트 NER로 자동 추출:
- 프로폴린스 경우: **오랄비·센소다인·콜게이트 플락스·테라브레스·어썸쿨 프로폴리스·광동 인후엔·리브러쉬 클러브·유시몰**
- 특히 **어썸쿨 프로폴리스 가글 600ml** = "프로폴리스" 계열 직접 경쟁자인데 우리 리스트에 없었음

**우리 프로젝트 반영**:
- [runner.py의 `parse_y2a`](experiments/runner.py) 보강: 사전 정의 경쟁사 + 응답 텍스트 NER 이중 추출
- jsonl 신규 필드: `other_brands_detected: [...]`
- 본실험 후 "정의된 N=6 외에 AI가 실제로 언급한 브랜드" 자동 리포트
- H14 (외부 증거) 관찰 변수로 직접 활용

### ★★★ B.2 쿼리 유형별 "맞춤 Y 지표" 설계

친구의 진짜 기여: **각 유형마다 고유 측정 지표** — Y2a를 단일 binary로 안 쓰고 유형에 맞게 분화.

**우리 반영 — Y2a 분화**:
- `Y2a_mention` (중립/CAT/SYM/PRC/USE): 멘션 여부
- `Y2a_positive` (BRD): 긍정 감성
- `Y2a_alternative` (COM): 대안 소환 + 미노출 여부
- `Y2a_wintieloss` (CMP): **{win, tie, loss} 3분** ← 친구의 핵심 아이디어
- `Y2a_singleselect` (DEC): 최종 선택 binary

→ 기존 Y2a "포괄적 정의" → **쿼리 유형별 특화 Y**. H5/H7 등 각 가설에 적합한 Y 사용 → 검정력·해석력 상승.

### ★★ B.3 "문제 / 원인 가설 / 바로 할 일" 3쌍 구조

친구의 자동 해석 템플릿:
```
1. 문제: 리스테린 대안 질문에서 미노출 100%
   원인 가설: AI가 후보로 인식 못 함
   바로 할 일: ✓ 비교 콘텐츠 ✓ 대안 키워드 리뷰 ✓ 차별 페이지 강화
```

**우리 반영**: [action_roadmap.md](consulting/action_roadmap.md) Before/After 섹션을 이 구조로 재구성. 발표 슬라이드도 결과 → 문제→원인→액션 3쌍으로.

### ★★ B.4 약점·강점 프롬프트 top 3 명시

친구 대시보드가 "전혀 언급 안 된 프롬프트 top 3" + "자주 언급된 top 3" 직접 표시.

**우리 반영**: h_tests.py에 추가. 본실험 후 **하위 10% Y2a 쿼리의 원문 + F 조합 리포트**. 발표 슬라이드 몰입도 ↑.

### ★★ B.5 경쟁사 × 지표 그리드

```
경쟁사              질문 수   대안 소환율   미노출률
리스테린            20        0.0%         100.0%
어썸쿨 프로폴리스    20        25.0%        70.0%
테라브레스          20        0.0%         100.0%
```

**우리 반영**: [eda.py](ml/scripts/eda.py)에 `competitor_x_metric_grid` 함수 추가. 경쟁사별 Y2a/Y5 한 화면.

### ★ B.6 쿼리 문구 — 구어체·반말 톤

친구: "구강청결제 처음 써보는데 뭐 사면 돼?" (짧은 구어체)
우리: "가정에서 쓸 수 있는 케겔운동기 추천해주세요." (정돈된 존댓말)

**문제**: 실제 ChatGPT 사용자 쿼리 분포는 전자에 가까움. 우리 쿼리가 "너무 공식적"이라 AI가 fact-sheet 응답만 생성할 위험.

**우리 반영**: [queries.yaml](experiments/prompts/queries.yaml) 24개 중 절반을 구어체로 교체. 유형당 3개 = (존댓말 긴 문장 1 + 반말 짧은 문장 1 + 불완전 문장 1). 쿼리 다양성 → H15 안정성 ↑.

- 친구: "현재 상태 + 액션 아이템"
- 우리: "왜 그런가 + 개선 시 예상 효과 크기"
- 통합: **실시간 대시보드 + 본실험 기반 예측**의 AiEO 컨설팅 풀 스택
