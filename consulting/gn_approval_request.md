## 1. 배경 — 왜 이 작업이 필요한가

### 1.1 시장 변화
| 신호 | 의미 |
|---|---|
| ChatGPT 주간 검색 사용자 8억 명+ (2026-Q1) | "검색 → 쇼핑"이 AI 채팅으로 이동 중 |
| Profound·Bluefish 등 GEO 측정 도구 11+ 등장 | "AI에 노출되는가"가 새 KPI |
| Schema.org Product·AggregateRating이 ChatGPT Shopping card 트리거 요건 | 페이지 구조화 데이터가 매출 직결 |
| 한국 요실금치료기 카테고리 전담 GEO 도구 글로벌 0개 | GN 컨설팅이 카테고리 첫 사례 가능 |

### 1.2 바디닥터 현재 진단 (실측)

자체 측정 점수 — sixthshop 4영역 배점 체계(35/25/20/20)를 차용해 17개 세부 지표를 자체 스크립트로 자동 산출.
*sixthshop 회사가 발표한 공식 점수가 아니라 같은 프레임워크로 우리가 직접 계산한 값입니다.*

| 영역 | 배점 | 바디닥터 | 약점 |
|---|---|---|---|
| Schema & Data Trust | 35 | **25** | aggregateRating 없음, 허가번호 identifier 없음 |
| Content & Intent Match | 25 | **13** | SPN 5 facet 부족, 근거 인용 0, 비교 콘텐츠 없음 |
| Media & Visual Signals | 20 | **6** | 이미지 alt 비율 35%만, 동영상·인포그래픽 없음 |
| Commerce & Eligibility | 20 | **15** | 리뷰 0개 |
| **총점** | 100 | **59** | — |

비교 (실측):
- 2080 SSG: 82/100 (이커머스 플랫폼 효과)
- 코웨이 테라솔: (크롤링 대기)
- 이지케이: 80/100대로 추정 (스펙·근거 강함)

**결론**: 바디닥터는 카테고리 평균 이하. 21점만 올려도 80점대 진입 가능 — 그 21점은 **모두 Wayne이 시행할 수 있는 페이지 단위 작업 + 리뷰 캠페인**.

## 2. 개선 실행 항목 — Wayne·연구팀이 시행할 것

각 항목 형식: **무엇을 / 왜 / 누가 / 필요한 GN 승인 / 예상 lift / 시점**

### 🔴 P0 — 페이지 직접 개선 (1~4주차)

#### 2.1 JSON-LD Product 풀 스키마 적용
- **무엇**: GN코스몰 바디닥터 PDP에 `<script type="application/ld+json">` 삽입. `@type=Product` + `aggregateRating` + `offers` + `identifier` + `brand`/`manufacturer` 풀세트
- **왜**: ChatGPT Shopping card 트리거 1순위 요건. Azoma·Profound 공통 우선순위 1. 현재 Product 최소형만 있어서 카드 렌더링 불가
- **누가**: Wayne이 스키마·코드 작성 ([초안 완성됨](diagnosis/bodydoctor_jsonld.json)) → GN 개발팀 또는 대행사가 PDP 템플릿에 삽입
- **GN 승인 필요**:
  1. GN코스몰 PDP 템플릿 수정 권한 (직접 또는 대행사 경유)
  2. **정확한 식약처 허가번호** (예: 제허 XX-XXX 호) — 스키마 `identifier` 필드
  3. **정확한 제조사 법인명** — `manufacturer.name` 필드 (식약처 DB에 GN/제너럴네트 미등록 → 화이트라벨 가능성 확인 필요)
- **예상 lift**: AI 멘션률 +30~60% (Ecomtent 고객 평균)
- **시점**: GN 정보 받은 직후 1주차

> 참고: `@type`은 Schema.org 표준에서 `MedicalDevice`도 가능하지만, 본 프로젝트는 의료광고법 회피 표기 원칙에 따라 **`Product`로 통일**합니다. SEO 측면 손실은 미미하며, 다른 필드(aggregateRating·offers·identifier)가 트리거 효과의 90% 이상을 담당합니다.

#### 2.2 인증 정보 상단 배너 + 본문 명시
- **무엇**: PDP Hero 직하에 "식약처 3등급 + 허가번호 제허 XX-XXX 호 + FDA Class 1·2 등록" 풀 텍스트 노출 + 본문 인증 섹션
- **왜**: AI는 페이지 상단 콘텐츠를 우선 파싱. 인증 정보가 하단·이미지 안에만 있으면 추천 신호로 작동 안 함. 본 연구 H5/H6(인증 명시가 추천 확률 ↑) 핵심 가설
- **누가**: Wayne이 카피 작성 ([초안 §2 완성됨](diagnosis/bodydoctor_pdp_copy_초안.md)) → GN PDP 적용
- **GN 승인 필요**:
  1. **FDA 등록번호** (Class 1과 Class 2 각각)
- **예상 lift**: 인증 풀 명시 +15~25% (업계 일반)
- **시점**: 1~2주차

#### 2.3 핵심 스펙 TABLE 구조화
- **무엇**: "99단계 강도, 5단계 자동 프로그램, 주파수 XX Hz, 사용 시간 15분/회" 등을 `<table>` 태그로 구조화 (BULLET·PARAGRAPH 대비 AI 파싱 정확도 ↑)
- **왜**: 본 연구 H1(HTML 포맷이 파싱 정확도에 영향) 가설의 직접 적용. AI는 TABLE 구조에서 스펙을 정확히 추출
- **누가**: Wayne 카피 → GN PDP 적용
- **GN 승인 필요**:
  - **정확한 스펙 수치** (현재 마케팅 카피에 모호하게 표기됨)
- **예상 lift**: 스펙 쿼리(PRC, USE) 추천률 +20%
- **시점**: 2주차

#### 2.4 Rufus SPN 5 facet 본문 삽입
- **무엇**: PDP 본문에 다음 5개 카테고리 자연스럽게 녹임:
  - **Subjective**: "간편한, 안정적인, 무소음"
  - **Event**: "출산 후 회복기, 갱년기 케어"
  - **Activity**: "앉아있는 동안 자동 운동, TV 시청 중 사용"
  - **Goal**: "골반저근 강화, 요실금 예방·관리"
  - **Audience**: "출산 후 여성, 40~60대 여성"
- **왜**: Amazon Rufus 논문 효과크기 ~2x. SYM/USE 쿼리 매칭률 직접 향상
- **누가**: Wayne 카피 ([초안 §4 완성됨](diagnosis/bodydoctor_pdp_copy_초안.md)) → GN PDP 적용
- **GN 승인 필요**: 페이지 수정 권한만 (정보 추가 불요)
- **예상 lift**: SYM/USE 쿼리 +50%
- **시점**: 2~3주차

#### 2.5 FAQ 섹션 + JSON-LD FAQPage 스키마
- **무엇**: PDP 하단에 5~10개 Q&A — "임산부 사용 가능?", "식약처 3등급 의미?", "이지케이와 차이?" 등 + FAQPage 스키마

- **누가**: Wayne 질문 초안 → GN 직원 인터뷰 후 답변 작성 → Wayne 검수 → 적용
- **GN 승인 필요**:

- **예상 lift**: FAQ형 쿼리(BRD-1, CMP) +25%

#### 2.6 비교 콘텐츠 ("바디닥터 vs 이지케이 비교표")
