# Glossary — 용어 사전

> 팀원/외부인이 모를 만한 본 프로젝트 용어들의 한 줄 정의.
> 알파벳 + 가나다 혼합. 새 용어는 처음 등장 시 추가.
> **팀 네이밍 규칙은 [팀_네이밍_규칙.md](팀_네이밍_규칙.md) 참고** — 이 글로서리는 마스터 정의, 네이밍 규칙은 cheat sheet + 룰.

---

## A

- **AiEO (AI Engine Optimization)**: AI 에이전트가 상품을 비교 평가할 때 우리 상품이 선택되게 만드는 SKU 단위 최적화. 본 프로젝트의 중심 개념.
- **API 응답 캐시**: `(model, prompt, seed)` 해시로 LLM 응답을 저장. 같은 호출 두 번 안 부르기 위함. `experiments/api_runs/_cache/`.

## C

- **클로즈드셋 실험**: AI에게 "이 N개 상품 중 골라라"라고 묻는 실험. 실제 AI 쇼핑 에이전트가 상품 DB를 받아 평가하는 구조와 일치.
- **컨텍스트 엔지니어링**: 프롬프트가 아니라, 모델이 보는 *모든 자산*(메모리, 지식베이스, 도구, 스타일가이드)을 설계하는 일.

## B

- **B1 / B2 / B3**: 데마 트랙 Phase B의 sub-phase. B1=관측형 본실험 1회차(1,920 호출, 2026-04-25 완료), B2=baseline 로지스틱 회귀, B3=XGBoost + SHAP.
- **BRD (브랜드 지명) 쿼리**: 쿼리 유형 중 우리 브랜드를 *직접 언급*하는 형태. 예: "바디닥터 효과 있나?". 8유형 중 1.

## C

- **CAT (카테고리 일반) 쿼리**: 쿼리 유형 중 카테고리 키워드만 사용. 예: "요실금치료기 추천해줘". 8유형 중 1.
- **CMP (비교) 쿼리**: 쿼리 유형 중 두 브랜드 직접 비교. 예: "바디닥터 vs 이지케이". 8유형 중 1.
- **COM (경쟁사 후 대안) 쿼리**: 쿼리 유형 중 경쟁사를 먼저 언급하고 대안 요청. 예: "이지케이 말고 다른 거". 8유형 중 1.

## D

- **DIGEST.md**: `docs/knowledge/DIGEST.md`. 프로젝트 현재 상태 한눈에. Ragless 핵심.
- **DEC (단일 선택) 쿼리**: 쿼리 유형 중 "딱 하나만 골라줘" 단일 선택을 강제. 8유형 중 1.

## F

- **F1 (HTML 포맷)**: TABLE / BULLET / PARAGRAPH 중 어느 형태로 페이지 정보를 제시하는지. 산공통 가설 H1.
- **F2 (JSON-LD 스키마 유무)**: none / Product / MedicalDevice 어느 수준. 산공통 가설 H2.
- **F3 (수치 구체성)**: explicit (구체수치) / ambiguous / none. 산공통 가설 H4.
- **F4 (인증 위치)**: top / bottom / none — 페이지 어디에 인증 표기. 산공통 가설 H5.
- **F5 (인증 상세)**: full / partial / none — 등급·허가번호 명시 정도. 산공통 가설 H5.
- **F6 (근거 유형)**: clinical (임상) / user_reviews (후기) / marketing (카피) / none. 산공통 가설 H7.
- **Findings**: `docs/knowledge/findings/` 디렉토리. 시간순 발견 누적.

## G

- **GEO (Generative Engine Optimization)**: 콘텐츠/페이지 레벨에서 AI 검색에 잘 잡히게 만드는 최적화. *발견*에 초점. AiEO와 구분됨.
- **GraphRAG**: 지식을 청크 더미가 아니라 엔티티-관계 그래프로 구축하는 검색 방식. 본 프로젝트는 안 씀(과함).

## H

- **H1**: HTML 포맷(F1)이 파싱 정확도 영향
- **H2**: JSON-LD(F2)가 파싱 정확도 영향
- **H3**: F1×F2 교호작용
- **H4**: 수치 구체성(F3)이 파싱 정확도 영향
- **H5**: 인증 명시(F4·F5)가 추천 확률 ↑
- **H6**: 인증 명시가 Y4 회피 ↓
- **H7**: 임상 인용(F6) > 사용자 후기
- **H8**: Y1 ≠ Y2 (GEO ≠ AiEO)
- **H9**: 경쟁 상황에서 구조화 효과 ↑
- **H10**: AI는 의료기기/비의료기기 구분 못 함 (사전등록 2026-04-23)
- **H11**: 쿼리에 "의료기기" 명시 시 카테고리 좁혀짐 (사전등록 2026-04-23)
- **H12**: Rufus SPN (USE 쿼리 × F6) 교호작용 (사전등록 2026-04-24)
- **~~H13~~**: 드롭됨 (Rich card rendering 파싱 한계)
- **H14**: 외부 증거(리뷰·인용·커뮤니티)가 추천 ↑ (관찰형, 사전등록 2026-04-24)
- **H15**: 쿼리 × F 교호작용 (사전등록 2026-04-24)

## J

- **JSON-LD**: 페이지 내 구조화 데이터 (Schema.org). Product / MedicalDevice 스키마 차이가 F2.

## L

- **L18 / L36**: 다구치(Taguchi) 직교 배열 표. 6요인 통제 실험 시 18 또는 36개 조합으로 압축.

## M

- **MedicalDevice 스키마**: Schema.org의 의료기기 전용 JSON-LD 타입. F2의 한 수준.
- **메모리 (Memory)**: `~/.claude/.../memory/`. Wayne 개인의 세션 간 영구 저장. 팀 공유 X. 지식베이스와 구분.

## P

- **Phase A**: 데이터 보강 (크롤링, 피처 추출) — 47 SKU + 18 브랜드 집계 완료
- **Phase B**: 관측형 AI 호출 + 데마 모델링 — B1 완료, B2/B3 진행 중
- **Phase C**: 바디닥터K 진단 + 컨설팅 산출물
- **Phase D**: GN 전달용 최종 산출물
- **Phase E**: 산공통 본실험 재개 (보류 해제 시)
- **Position bias**: AI에게 N개 상품을 보여줄 때 *순서*가 선택에 영향을 주는 편향. 매 반복 셔플로 통제 (R4).
- **PRC (가격·예산) 쿼리**: 쿼리 유형 중 가격·예산 명시. 예: "200만원 예산". 8유형 중 1.

## Q

- **Q1, Q2**: 쿼리 수준 요인. Q1=쿼리 유형(정량/정성/비교/DECISION), Q2=조건 복잡도.

## R

- **R1 (반복 횟수)**: 본실험 호출 15~20회 강제 — 변동성 추정용
- **R2 (temperature)**: API 기본값 (0.7 추정) 유지. **0 금지** (실제 사용자 환경 시뮬레이션 위함)
- **R3 (모델 버전 고정)**: 결과 jsonl에 모델 버전 메타데이터 포함
- **R4 (Position bias 셔플)**: 매 반복마다 상품 리스트 셔플
- **R5 (랜덤 시드 기록)**: 셔플·샘플링 시드 모두 jsonl 기록
- **R6 (Y변수 이중 정의)**: binary Y + continuous Y 둘 다 산출
- **R7 (이항 비율 95% CI)**: Wilson score CI 함께 표기
- **R8 (Raw 응답 보존)**: API 응답 원문 + 메타데이터 jsonl 저장
- **R9 (파일럿 우선)**: 본실험 전 20~30 조합 × 5회 파일럿
- **R10 (버티컬 분리)**: 의료기기와 가글을 같은 모델로 묶지 않음
- **Ragless**: 매번 검색하지 않고, 미리 컴파일된 구조화 지식 레이어를 모델이 직접 읽는 방식. 본 프로젝트가 채택.
- **RAG (개선된)**: 청크+벡터DB 기반 검색. 본 프로젝트는 안 씀(코퍼스 작음).

## S

- **SHAP**: 머신러닝 모델 설명 기법. 피처 기여도 계산. `shap` 패키지.
- **So What 슬라이드**: 발표 마무리에 "그래서 결론적으로 바디닥터K 추천율이 X%→Y%"를 보여주는 슬라이드.
- **SYM (증상 기반) 쿼리**: 쿼리 유형 중 증상으로 시작. 예: "산후 요실금에 뭐 좋아?". 8유형 중 1.

## U

- **USE (사용 시나리오) 쿼리**: 쿼리 유형 중 페르소나·상황 위주. 예: "출산 후 짬짬이 쓸 수 있는". 8유형 중 1.

## W

- **워크플로우 (Workflow)**: 반복되는 작업 절차. `docs/workflows/`에 템플릿화.

## Y

- **Y1~Y4**: 종속변수. Y1=파싱 정확도, Y2=추천 선택률, Y3=근거 수치 기반도, **Y4=AI 안전성 회피 반응** (의료기기 전용).

## 가나다

- **가설 사전 등록**: 데이터 보기 전에 검정할 가설을 명시. p-hacking 방지.
- **갱신 의식**: 매주 금요일 30분 지식베이스 정리. APPLICATION.md §2.5, knowledge/README.md §4.
- **버티컬 (Vertical)**: 의료기기 / 가글 같은 제품 카테고리. 버티컬 간 추천 드라이버가 다름이 핵심 발견.
- **본실험**: 파일럿 후 본격 실행. 15~20회 반복.
- **파일럿 (Pilot)**: 본실험 전 작은 규모로 변동성 측정. R9.
- **컨설팅 산출물**: 학기 프로젝트 결과물을 GN그룹 컨설팅용으로 전환한 것. `consulting/`.
