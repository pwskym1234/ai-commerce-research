# 직교배열 L36 — 본실험 가상 페이지 설계 표

## 0. 결정

본실험 산공통 가상 페이지는 **L36 직교배열**로 설계한다 (2026-04-23 확정).

## 1. 배경 — 왜 직교배열을 쓰나

우리 실험은 6요인:
- F1 HTML 포맷 (3수준: TABLE/BULLET/PARAGRAPH)
- F2 JSON-LD (3수준: Product/MedicalDevice/없음)
- F3 수치 구체성 (2수준: 정확/모호)
- F4 인증 정보 위치 (3수준: 상단상세/하단간략/없음)
- F5 인증 명시 수준 (3수준: 풀텍스트/인증완료/없음)
- F6 임상/효과 근거 (3수준: 임상/후기/없음)

**전체 조합 (풀 팩토리얼)** = 3×3×2×3×3×3 = **486개**
- 본실험 호출 수 = 486 × 4쿼리 × 20반복 × 3 LLM = **116,640회**, 예상 비용 ~$1,200, 페이지 작업 ~50시간 — **비현실적**

**직교배열의 의미**: 각 요인의 각 수준이 다른 요인의 각 수준과 *균등하게* 짝지어진 부분집합을 표준화한 표 (다구치). 균형 구조 덕분에 각 요인 효과 추정 시 다른 요인 영향이 평균에서 캔슬됨.

## 2. 비교 옵션 정량 분석

| 설계 | 조합 | 본실험 호출 | 예상 비용 (USD) | 페이지 작업 | 검정 가능 |
|------|------|------------|----------------|-----------|-----------|
| L18 | 18 | 4,320 | ~$43 | ~3시간 | 모든 주효과 (F1~F6). H3(F1×F2) ❌ |
| L26 (L18 + F1×F2 보충) | 26 | 6,240 | ~$62 | ~4시간 | 주효과 + H3. F4×F6 등 ❌ |
| **L36** ★ | **36** | **8,640** | **~$86** | **~6시간** | **주효과 + 대부분 2차 교호작용 (H3 포함)** |
| 풀팩토리얼 | 486 | 116,640 | ~$1,200 | ~50시간 | 모든 효과 + 모든 교호작용 (3차+) |

(비용: OpenAI gpt-4o 기준 호출당 ~$0.01, 4쿼리 × 20반복 × 3 LLM 가정)

## 3. L36 채택 정당화

**효용**:
- ✅ 모든 주효과 검정 가능 (H1, H2, H4, H5, H6, H7)
- ✅ 2차 교호작용 검정 가능 (H3 = F1×F2 포함)
- ✅ Wayne의 모든 사전 등록 가설 H1~H11 본실험 1회로 검정 가능

**비용 단점**:
- L18 대비 +$43, +3시간 작업 — **무시할 수준**
- 풀팩토리얼 대비 1/13 — **충분히 압축**

**학술 표준**:
- 다구치 방법은 산업공학 통계의 표준. 발표에서 정당화 쉬움
- L36은 가장 흔히 쓰이는 직교배열 중 하나 (Phadke 1989, Roy 2010)
- 학술적으로 "왜 직교배열?", "왜 L36?" 둘 다 표준 교과서 인용으로 답변 가능

## 4. 우리 6요인 → L36 매핑

L36은 표준 형태로 13컬럼 (각 컬럼이 하나의 요인 수준 할당). 우리는 6요인만 쓰므로 13컬럼 중 6개만 사용 + 나머지는 dummy/오차 추정.

### 매핑 권고
| L36 컬럼 | 우리 요인 | 수준 |
|---------|---------|------|
| Col 1 | F1 HTML 포맷 | TABLE / BULLET / PARAGRAPH |
| Col 2 | F2 JSON-LD | Product / MedicalDevice / 없음 |
| Col 3 | F3 수치 구체성 | 정확 / 모호 (3수준 표인데 2수준 매핑 시 한 수준 더미) |
| Col 4 | F4 인증 위치 | 상단상세 / 하단간략 / 없음 |
| Col 5 | F5 인증 상세 | 풀텍스트 / 인증완료 / 없음 |
| Col 6 | F6 근거 | 임상 / 후기 / 없음 |
| Col 7~13 | 미사용 (오차 추정) | — |

### F3가 2수준이라 약간의 불균형
L36은 본래 모든 컬럼이 3수준이라 가정. F3만 2수준이면 균형이 깨질 수 있음. 두 가지 처리:
- 옵션 A: F3을 3수준으로 확장 (정확 / 부분명시 / 모호)
- 옵션 B: F3를 별도 트랙으로 빼고 풀팩토리얼 (3×3×3×3×3 = 243의 부분 → L27 또는 다른 표 사용)

→ **옵션 A 권장**: F3을 3수준으로 확장하면 L36 그대로 사용 가능. 의미적으로도 "정확/부분명시/모호"가 자연스러움.

## 5. 실제 생성 (구현 단계)

```python
import pyDOE2

# L36 표는 pyDOE2/DoE.base에 표준 표로 포함
design = pyDOE2.gsd(levels=[3, 3, 3, 3, 3, 3], reduction=13)  # 36/468 ≈ 1/13
# 또는 DoE.base/Minitab에서 L36(3^13) 호출
```

세부 구현은 가상 페이지 제작 시점에 워크플로우 [docs/workflows/](../../workflows/)에 추가.

## 6. 한계

- 3차 이상 교호작용 검정 불가 — 보통 무시 가능. 학술 표준
- L36은 정통 Taguchi 표에서 일부 컬럼이 다른 컬럼들과 부분 confound — confounding pattern 사전 확인 필요
- 가설이 **사전 등록되어야** 직교 분석의 정직한 p-value 사용 가능. 사후 가설 검정 시 별도 표시
